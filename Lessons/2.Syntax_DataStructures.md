## Part 1 — Comments: Documenting Your Code

A comment in R is any line (or part of a line) that begins with `#`. R completely ignores everything after the `#` symbol — it is written for humans, not the computer.

Comments are one of the most important habits you can build as a programmer. Good comments explain *why* something is done, not just *what* is being done. When you return to code you wrote six months ago — or when someone else reads your code — comments are the difference between understanding it instantly and being completely lost.

```r
# This is a full-line comment — R ignores this entirely

x <- 42        # This is an inline comment — the code runs, then R ignores the rest

# You can use comments to temporarily disable a line of code:
# mean(my_data)   <-- this will NOT run
```

### Documenting a block of code

It is good practice to use comments to label sections of your script so it reads like a document. A common convention is to use a row of dashes or equals signs to create visual separators.

```r
# ============================================================
# SECTION 1 — Load and clean the data
# ============================================================

# Read in the penguin survey data from CSV
penguins <- read.csv("penguins.csv")

# Remove rows with missing values before analysis
penguins_clean <- na.omit(penguins)


# ============================================================
# SECTION 2 — Summary statistics
# ============================================================

# Calculate the mean body mass across all penguins
mean(penguins_clean$body_mass_g)
```

### Documenting a function

When you write a function (covered in a later lecture), comments at the top explain what it does, what inputs it expects, and what it returns. This is how all well-written R packages document their code.

```r
# calculate_bmi()
# -------------------------------------------------------
# Calculates a simple size index for penguins.
#
# Arguments:
#   mass_g       — body mass in grams (numeric)
#   flipper_mm   — flipper length in millimetres (numeric)
#
# Returns:
#   A numeric value: mass divided by flipper length
# -------------------------------------------------------
calculate_bmi <- function(mass_g, flipper_mm) {
  mass_g / flipper_mm
}
```

This style of commenting makes your code self-documenting — anyone reading it knows exactly what the function expects and what it produces, without needing to read through the logic.

## Part 2 — Variables and the Assignment Operator `<-`

### What is a variable?

A **variable** is a named container that stores a value. Once you assign a value to a variable, you can refer to it by name anywhere in your script. Variables are the fundamental building block of any program — they let you store results, pass data between steps, and avoid repeating yourself.

### The assignment operator `<-`

In R, you assign a value to a variable using `<-`. Read it as *"gets"* — the variable on the left **gets** the value on the right.

```r
age <- 25
```

Read this as: *"age gets 25"*. Now whenever you type `age`, R knows it means 25.

```r
age <- 25
name <- "Alice"
height_cm <- 168.5
is_student <- TRUE

# Print them
age          # 25
name         # "Alice"
height_cm    # 168.5
is_student   # TRUE
```

### Why `<-` and not `=`?

You *can* use `=` to assign variables in R and it will often work, but `<-` is the standard R convention and is strongly preferred. The reason matters and is explained in the next section — `=` has a different, specific job inside function calls.

```r
# Both of these work for assignment:
x <- 10     # preferred — use this
x = 10      # works, but avoid at the top level

# Use <- for variables, = for function arguments (see next section)
```

### Naming rules for variables

Variable names in R must follow a few simple rules:

- Can contain letters, numbers, `.` and `_`
- Must **start** with a letter or `.`
- Cannot contain spaces
- R is **case-sensitive**: `Age` and `age` are two completely different variables

```r
# Good variable names
body_mass <- 4200
bill.length <- 39.1
sampleSize <- 50        # camelCase style
sample_size <- 50       # snake_case style — most common in R

# Bad variable names (these will cause errors)
# 2ndSample <- 50       # cannot start with a number
# body mass <- 4200     # cannot contain a space
# my-variable <- 10     # hyphens are not allowed
```

> **Convention:** Most modern R code uses `snake_case` — all lowercase with underscores between words. This is the style used by the tidyverse (ggplot2, dplyr, etc.) and is the recommended approach.

## Part 3 — The `=` Sign Inside Functions

The `=` sign has a specific, separate role from `<-`. Inside a function call, `=` is used to pass **named arguments** — it connects an argument name to the value you want to give it.

### What is a function argument?

A function is a named action that does something. Most functions accept inputs, called **arguments**, that control how the function behaves. You pass values to arguments using `=` inside the brackets.

```r
# round() takes two arguments:
#   x      — the number to round
#   digits — how many decimal places to keep

round(x = 3.14159, digits = 2)    # returns 3.14
```

Here `=` is not creating a variable — it is saying *"for the argument named `digits`, use the value 2"*. This `=` lives only inside the function call.

```r
# seq() generates a sequence of numbers
seq(from = 1, to = 10, by = 2)    # 1 3 5 7 9

# paste() joins text together
paste("Hello", "world", sep = ", ")   # "Hello, world"

# mean() with the na.rm argument
scores <- c(88, 92, NA, 75, 85)
mean(scores, na.rm = TRUE)            # ignores the NA, returns 85
```

### The key distinction

```r
# <- creates a variable in your environment
result <- 42          # 'result' now exists as a variable

# = passes a value to a named argument inside a function
round(3.14159, digits = 2)    # 'digits' is NOT a new variable — it lives inside round()
```

This is why keeping them separate matters. If you use `=` for everything it becomes ambiguous — is this creating a variable or passing an argument? Using `<-` for assignment and `=` for arguments makes your code unambiguous and readable at a glance.

## Part 4 — Data Types and Data Structures

R has several ways of organising and storing data. The four structures you will use most often are **vectors**, **data frames**, and **lists**. Understanding the difference between them is essential — the right structure makes your analysis easier; the wrong one creates unnecessary complexity.

---

## 4A — Vectors

A vector is the simplest and most fundamental data structure in R. It is an **ordered collection of values of the same type**. Every value in a vector must be the same data type — all numbers, all text, or all logical values.

You create a vector using `c()`, which stands for *combine* or *concatenate*.

### Numeric vector

```r
# Body masses of five penguins in grams
body_mass <- c(3750, 3800, 4200, 4600, 5200)

body_mass          # 3750 3800 4200 4600 5200
length(body_mass)  # 5 — how many elements
```

### Character vector

```r
# Species names
species <- c("Adelie", "Adelie", "Chinstrap", "Gentoo", "Gentoo")

species          # "Adelie" "Adelie" "Chinstrap" "Gentoo" "Gentoo"
```

### Logical vector

```r
# Did each penguin exceed 4500g?
heavy <- c(FALSE, FALSE, FALSE, TRUE, TRUE)
```

### What happens if you mix types?

If you try to mix types, R will silently **coerce** everything to the most flexible type. This can cause subtle bugs if you are not aware of it.

```r
mixed <- c(1, 2, "three", 4)
mixed   # "1" "2" "three" "4"  — all became text (character)
```

### Working with vectors

Vectors are the engine of R. Most of R's built-in functions operate on vectors directly:

```r
flipper <- c(181, 186, 195, 213, 210, 202, 198)

mean(flipper)     # average
median(flipper)   # middle value
sd(flipper)       # standard deviation
sum(flipper)      # total
min(flipper)      # smallest
max(flipper)      # largest
length(flipper)   # how many elements
```

### Vectorised operations

One of R's most powerful features is that arithmetic on vectors applies **element by element** automatically, without any loops:

```r
bill_mm <- c(39.1, 39.5, 40.3, 36.7, 39.3)

# Convert all values from mm to cm in one operation
bill_cm <- bill_mm / 10
bill_cm    # 3.91 3.95 4.03 3.67 3.93

# Add 5 to every value
bill_mm + 5    # 44.1 44.5 45.3 41.7 44.3
```

### Accessing elements with indexing

You access individual elements of a vector using square brackets `[]`. R uses **1-based indexing** — the first element is at position 1, not 0.

```r
species <- c("Adelie", "Chinstrap", "Gentoo")

species[1]        # "Adelie"    — first element
species[3]        # "Gentoo"    — third element
species[c(1, 3)]  # "Adelie" "Gentoo"  — first and third
species[-2]       # "Adelie" "Gentoo"  — everything except the second
```

### Named vectors

You can give each element a name, making your code more readable:

```r
island_counts <- c(Biscoe = 168, Dream = 124, Torgersen = 52)

island_counts["Biscoe"]    # 168
island_counts["Dream"]     # 124
```

## 4B — Data Frames

A data frame is R's version of a **table** — rows and columns, just like a spreadsheet. It is the most common structure you will work with for real data analysis. Each column is a vector, and all columns must have the same number of rows. Crucially, different columns *can* hold different data types — this is what makes data frames more flexible than plain vectors.

### Creating a data frame

```r
penguins_df <- data.frame(
  species      = c("Adelie", "Adelie", "Chinstrap", "Gentoo"),
  island       = c("Torgersen", "Torgersen", "Dream", "Biscoe"),
  bill_length  = c(39.1, 39.5, 46.5, 50.0),
  body_mass    = c(3750, 3800, 3500, 5700),
  is_juvenile  = c(FALSE, FALSE, TRUE, FALSE)
)

penguins_df
```

This creates a table with 4 rows and 5 columns, where `species` and `island` are character columns, `bill_length` and `body_mass` are numeric, and `is_juvenile` is logical — all different types living together.

### Exploring a data frame

These are the first functions you should run on any new dataset:

```r
nrow(penguins_df)      # number of rows: 4
ncol(penguins_df)      # number of columns: 5
dim(penguins_df)       # dimensions: 4 5
names(penguins_df)     # column names
head(penguins_df)      # first 6 rows
tail(penguins_df)      # last 6 rows
str(penguins_df)       # structure — types of each column
summary(penguins_df)   # summary statistics for each column
```

### Accessing columns with `$`

The `$` operator is the most common way to pull out a single column. It returns the column as a vector.

```r
penguins_df$species       # "Adelie" "Adelie" "Chinstrap" "Gentoo"
penguins_df$body_mass     # 3750 3800 3500 5700

# You can then use vector functions on it directly
mean(penguins_df$body_mass)    # 4187.5
max(penguins_df$bill_length)   # 50.0
```

### Accessing rows and columns with `[row, column]`

Data frames use two-dimensional indexing: `[row, column]`. Leaving one dimension blank means *"all of them"*.

```r
penguins_df[1, ]        # entire first row
penguins_df[, 3]        # entire third column (bill_length)
penguins_df[2, 3]       # row 2, column 3: 39.5
penguins_df[1:2, ]      # first two rows
penguins_df[, "species"]  # column by name
```

### Filtering rows

One of the most common operations — keeping only rows that meet a condition:

```r
# Keep only Gentoo penguins
gentoo <- penguins_df[penguins_df$species == "Gentoo", ]

# Keep only penguins with body mass over 4000g
heavy <- penguins_df[penguins_df$body_mass > 4000, ]

# Keep only adult penguins
adults <- penguins_df[penguins_df$is_juvenile == FALSE, ]
```

### Adding a new column

Assigning to a new column name creates it automatically:

```r
# Convert body mass from grams to kilograms
penguins_df$body_mass_kg <- penguins_df$body_mass / 1000

penguins_df$body_mass_kg    # 3.75 3.80 3.50 5.70
```

## 4C — Lists

A list is the most flexible data structure in R. Unlike a vector (which must contain values of the same type) or a data frame (which must have equal-length columns), a list can contain **anything** — vectors, data frames, other lists, numbers, text, model outputs — all mixed together, all of different lengths.

Think of a list as a filing cabinet. Each drawer can hold something completely different.

### Creating a list

```r
# A list holding different types of information about one penguin
penguin_record <- list(
  species     = "Gentoo",
  island      = "Biscoe",
  measurements = c(bill = 50.0, flipper = 213, mass = 5700),
  is_juvenile = FALSE,
  notes       = "Observed near colony edge. Very large individual."
)
```

This single list contains a character string, another character string, a named numeric vector, a logical value, and a longer character string — five completely different things.

### Accessing list elements

Lists can be accessed in three ways:

```r
# 1. By name using $ (most readable)
penguin_record$species          # "Gentoo"
penguin_record$measurements     # bill=50.0 flipper=213 mass=5700

# 2. By name using [[ ]] (same result as $)
penguin_record[["island"]]      # "Biscoe"

# 3. By position using [[ ]]
penguin_record[[3]]             # the measurements vector

# Access an element inside a nested element
penguin_record$measurements["bill"]    # 50.0
```

The difference between `[ ]` and `[[ ]]` is important: single brackets `[ ]` return a **list** containing that element; double brackets `[[ ]]` return the **element itself**. For most purposes you want `[[ ]]` or `$`.

```r
penguin_record[1]     # returns a list of length 1
penguin_record[[1]]   # returns "Gentoo" directly
```

### A practical example — storing analysis results

Lists are especially useful for storing the results of an analysis where different outputs have different shapes:

```r
# Run a t-test comparing body mass between two species
adelie_mass   <- c(3750, 3800, 3250, 4050, 3450)
chinstrap_mass <- c(3500, 3900, 3650, 3525, 3725)

test_result <- t.test(adelie_mass, chinstrap_mass)

# test_result is a list — inspect it
str(test_result)

# Access specific parts
test_result$p.value      # the p-value
test_result$statistic    # the t-statistic
test_result$conf.int     # the confidence interval
```

This is how R stores nearly all statistical model outputs — as a list of components that you can extract individually.

### A list of data frames

Lists become particularly powerful when you need to store multiple datasets together:

```r
# Store data for three islands as separate data frames inside one list
island_data <- list(
  Biscoe    = data.frame(species = c("Gentoo", "Adelie"),
                          mass    = c(5700, 3750)),
  Dream     = data.frame(species = c("Chinstrap", "Adelie"),
                          mass    = c(3500, 3200)),
  Torgersen = data.frame(species = c("Adelie"),
                          mass    = c(4050))
)

# Access the Biscoe data frame
island_data$Biscoe

# Access the mass column within the Biscoe data frame
island_data$Biscoe$mass     # 5700 3750
```

## Summary

### The assignment and argument operators

| Symbol | Used for | Example |
|---|---|---|
| `<-` | Assigning a value to a variable | `x <- 42` |
| `=` | Passing a value to a function argument | `round(3.14, digits = 1)` |
| `#` | Writing a comment (R ignores this) | `# this is a note` |

### Data structures at a glance

| Structure | What it holds | Same type required? | Dimensions |
|---|---|---|---|
| **Vector** | A sequence of values | Yes — all must match | 1D |
| **Data frame** | A table (rows × columns) | Per column (columns can differ) | 2D |
| **List** | Anything — mixed types and sizes | No | Flexible |

### When to use each

Use a **vector** when you have a single sequence of values of the same type — a column of measurements, a list of species names, a set of TRUE/FALSE flags.

Use a **data frame** when you have tabular data with multiple variables measured on the same observations — this is the standard format for any dataset you would analyse or plot.

Use a **list** when your data is irregular or mixed — storing model outputs, combining datasets of different sizes, or grouping related objects that do not fit neatly into a table.
