## 1. Understanding R Syntax and Conventions

### What is Syntax?

**Syntax** is the set of rules that defines how code must be written for R to understand it. Just as English sentences follow grammar rules, R code follows syntax rules. Breaking these rules produces errors; following them lets R execute your instructions correctly.

### Expressions and Statements

The simplest thing you can do in R is type an **expression** — something R can evaluate and return a result for.

```r
# Arithmetic expressions — R as a calculator
2 + 3          # 5
10 / 4         # 2.5
3 ^ 2          # 9  (exponentiation)
sqrt(16)       # 4
abs(-7)        # 7  (absolute value)
```

A **statement** typically does something more — like storing a result, calling a function, or controlling the flow of a program.

### The Assignment Operator `<-`

In R, you store values using the **assignment operator** `<-`. This creates a named object (a **variable**) that holds a value.

```r
# Storing values in variables
x <- 42
name <- "Alice"
passed <- TRUE

# The value on the right is stored in the name on the left
result <- 10 * 3 + 5
result        # Type the name to print its value: 55
```

> **`<-` vs `=`:** Both work for assignment in most situations, but `<-` is the conventional and strongly preferred style in R. Reserve `=` for passing arguments inside function calls (e.g., `mean(x, na.rm = TRUE)`).

**Keyboard shortcut:** In RStudio, press `Alt + -` (Windows/Linux) or `Option + -` (Mac) to insert `<-`.

### Printing Output

R automatically prints the result of any expression typed in the console. When writing scripts, use `print()` to explicitly display output.

```r
x <- 100

x           # Auto-prints in the console: [1] 100
print(x)    # Explicit print — same result, better in scripts

# What does [1] mean?
# It's the index of the first element on that line — not part of the value.
# For long vectors, you'll see [1], [20], [39], etc. to help you track position.
```

### Comments

A **comment** is text that R ignores completely. Use `#` to start a comment. Comments are essential for explaining your code to others — and to your future self.

```r
# This is a full-line comment — R skips it entirely

x <- 5    # This is an inline comment — code runs, then comment is ignored

# Comments are your best tool for readable, maintainable code.
# Describe WHY you are doing something, not just WHAT.

# Bad comment:
x <- x * 2    # multiply x by 2

# Good comment:
x <- x * 2    # Double the dosage to convert from mg to mcg
```

### Naming Conventions

Variable names in R must follow these rules:

- Must **start with a letter** (not a number or symbol)
- Can contain **letters, numbers, dots `.`, and underscores `_`**
- Are **case-sensitive**: `Age`, `age`, and `AGE` are three different variables
- Cannot use **reserved words** like `if`, `else`, `TRUE`, `FALSE`, `for`, `function`

```r
# Valid names
patient_age <- 45
bmi.score   <- 27.4
group1      <- "control"
.hidden     <- TRUE      # starts with a dot — valid but unusual

# Invalid names — these will cause errors
# 1st_reading <- 120    # starts with a number
# my-variable <- 5      # hyphens not allowed
# TRUE <- 1             # reserved word
```

**Naming style conventions** in R:

| Style | Example | Notes |
|-------|---------|-------|
| `snake_case` | `blood_pressure` | Most common in tidyverse — **recommended** |
| `dot.case` | `blood.pressure` | Common in base R |
| `camelCase` | `bloodPressure` | Common in other languages, used in some R packages |

> **Recommendation:** Use `snake_case` consistently throughout your scripts. It is the most readable and the style used throughout this course.

### Spacing and Readability

R ignores extra spaces, but writing readable code is a professional habit worth building early.

```r
# Hard to read
x<-5*3+sqrt(16)/2

# Easy to read — same result
x <- 5 * 3 + sqrt(16) / 2

# Conventions:
# - Spaces around operators: x <- y + z  (not x<-y+z)
# - Space after commas: c(1, 2, 3)  (not c(1,2,3))
# - No space before a comma or before parentheses: mean(x)  (not mean (x))
```

### Functions — A First Look

A **function** is a named operation that takes **inputs** (called **arguments**) and returns an **output**. You've already used a few: `sqrt()`, `abs()`, `print()`.

```r
# General form:
# function_name(argument1, argument2, ...)

round(3.14159, digits = 2)    # 3.14
nchar("hello")                 # 5 (number of characters)
paste("R", "is", "great")     # "R is great"
```

Arguments can be passed **by position** or **by name**:

```r
round(3.14159, 2)              # By position: value, then digits
round(x = 3.14159, digits = 2) # By name: same result, more explicit
round(digits = 2, x = 3.14159) # Named arguments can be in any order
```

### Getting Help

```r
# Open the help page for a function
?mean
?round
help("sqrt")

# Search for help if you don't know the exact function name
??histogram
help.search("normal distribution")

# See example usage
example(mean)
```

## 2. Introduction to R's Core Data Types

Every value in R has a **type** (also called a **class**). The type determines what operations are valid and how R stores the value in memory. Understanding types prevents errors and ensures your analyses behave as expected.

R has four data types you will use constantly:

| Type | Stores | Example Values |
|------|--------|----------------|
| `numeric` | Numbers (decimals and integers) | `3.14`, `42`, `-7.5` |
| `character` | Text / strings | `"hello"`, `"Patient A"` |
| `logical` | Boolean truth values | `TRUE`, `FALSE` |
| `factor` | Categorical groups with fixed levels | `"low"`, `"medium"`, `"high"` |

You can always check the type of any object using `class()` or `typeof()`:

```r
class(3.14)       # "numeric"
class("hello")    # "character"
class(TRUE)       # "logical"
class(factor("low"))  # "factor"
```

## 3. Numeric Data

### What is Numeric Data?

Numeric data represents **quantities** — anything you can measure or count. This is the most common data type in scientific and analytical work.

R has two sub-types of numeric:

- **double** (the default): stores decimal numbers with floating-point precision
- **integer**: stores whole numbers only, written with an `L` suffix

```r
# Double (default numeric)
height    <- 175.5
weight    <- 68.2
pi_approx <- 3.14159

# Integer
n_patients <- 30L
year       <- 2024L

typeof(height)      # "double"
typeof(n_patients)  # "integer"
class(height)       # "numeric"
class(n_patients)   # "integer"
```

In practice, you rarely need to distinguish between double and integer — R handles the difference automatically in most situations. Use `L` only when a function specifically requires an integer or when you want to save memory with large whole-number datasets.

### Arithmetic with Numerics

```r
a <- 15
b <- 4

a + b    # Addition:       19
a - b    # Subtraction:    11
a * b    # Multiplication: 60
a / b    # Division:       3.75
a %% b   # Modulo:         3  (remainder after division)
a %/% b  # Integer divide: 3  (floor of a/b)
a ^ b    # Exponentiation: 50625
```

### Useful Numeric Functions

```r
x <- c(3.7, 1.2, 8.5, 4.1, 6.9)

round(x, 1)       # Round to 1 decimal: 3.7 1.2 8.5 4.1 6.9
floor(x)          # Round down:         3   1   8   4   6
ceiling(x)        # Round up:           4   2   9   5   7
trunc(x)          # Truncate (drop decimals): 3 1 8 4 6

sqrt(64)          # 8
log(100)          # Natural log: 4.605...
log10(100)        # Log base 10: 2
log(8, base = 2)  # Log base 2:  3
exp(1)            # e^1: 2.718...
```

### Checking and Testing Numeric Values

```r
x <- 42.5

is.numeric(x)     # TRUE
is.integer(x)     # FALSE
is.finite(x)      # TRUE
is.infinite(x)    # FALSE
is.nan(x)         # FALSE (NaN = Not a Number, e.g. 0/0)
is.na(x)          # FALSE (NA = missing value)
```

Special numeric values to be aware of:

```r
1 / 0       # Inf   (positive infinity)
-1 / 0      # -Inf  (negative infinity)
0 / 0       # NaN   (not a number — undefined result)
NA_real_    # NA    (missing numeric value)
```

### Converting to Numeric

```r
as.numeric("3.14")    # 3.14   (character to numeric)
as.numeric(TRUE)      # 1      (logical to numeric)
as.numeric(FALSE)     # 0
as.numeric("hello")   # NA with a warning — "hello" has no numeric equivalent
```

## 4. Character Data

### What is Character Data?

Character data (also called **strings**) stores **text**. Any value enclosed in single `'...'` or double `"..."` quotation marks is a character. Always prefer double quotes as a convention.

```r
first_name   <- "Alice"
species_name <- "Homo sapiens"
site_code    <- "SITE_04B"
dob          <- "1990-07-23"   # Dates stored as text before conversion
empty_string <- ""             # An empty string — still character type
```

### Key Character Functions

```r
x <- "Data Analysis with R"

nchar(x)                    # Number of characters: 21
toupper(x)                  # "DATA ANALYSIS WITH R"
tolower(x)                  # "data analysis with r"
substr(x, 1, 4)             # "Data"  (extract characters 1 to 4)
gsub("a", "@", x)           # Replace all "a" with "@"
sub("a", "@", x)            # Replace only the FIRST "a"
trimws("  hello  ")         # "hello"  (remove leading/trailing spaces)
trimws("  hello  ", "left") # "hello  " (remove left spaces only)
```

### Combining Strings with `paste()`

```r
first <- "Alice"
last  <- "Smith"

paste(first, last)             # "Alice Smith"  (space separator by default)
paste(first, last, sep = "_")  # "Alice_Smith"
paste0(first, last)            # "AliceSmith"   (no separator)

# Combining a vector of strings into one
words <- c("The", "quick", "brown", "fox")
paste(words, collapse = " ")   # "The quick brown fox"
```

**Research example:** Generating participant IDs automatically:

```r
site   <- "LAB"
number <- c(1, 2, 3, 4, 5)
ids    <- paste0(site, "_", sprintf("%03d", number))
ids
# [1] "LAB_001" "LAB_002" "LAB_003" "LAB_004" "LAB_005"
```

### Searching Within Strings

```r
x <- c("control_A", "treatment_B", "control_B", "treatment_A")

grepl("control", x)       # TRUE FALSE TRUE FALSE  (logical: contains "control"?)
grep("control", x)        # 1 3  (positions where pattern is found)
grep("control", x, value = TRUE)  # "control_A" "control_B"  (return matches)
```

### Checking and Converting Character Data

```r
is.character("hello")    # TRUE
is.character(42)         # FALSE

as.character(3.14)       # "3.14"
as.character(TRUE)       # "TRUE"
as.character(2024L)      # "2024"
```

> **Important:** Numbers stored as character strings look like numbers but behave like text. `"42" + 1` will produce an error. Always convert with `as.numeric()` before doing arithmetic.

```r
x <- "42"
x + 1                  # Error: non-numeric argument to binary operator
as.numeric(x) + 1      # 43
```

## 5. Logical Data

### What is Logical Data?

Logical data can only take two values: **`TRUE`** or **`FALSE`**. These are R's built-in Boolean values and must always be written in full capitals.

```r
passed_exam   <- TRUE
has_missing   <- FALSE
above_average <- TRUE
```

Logical values are most commonly produced as the **result of a comparison**:

```r
10 > 5      # TRUE
3 == 3      # TRUE  (equality uses ==, not =)
7 != 7      # FALSE (not equal)
4 >= 10     # FALSE
"cat" == "dog"  # FALSE
```

### Comparison Operators

| Operator | Meaning | Example | Result |
|----------|---------|---------|--------|
| `==` | Equal to | `5 == 5` | `TRUE` |
| `!=` | Not equal to | `5 != 3` | `TRUE` |
| `>` | Greater than | `7 > 4` | `TRUE` |
| `<` | Less than | `2 < 1` | `FALSE` |
| `>=` | Greater than or equal | `5 >= 5` | `TRUE` |
| `<=` | Less than or equal | `3 <= 2` | `FALSE` |

> **Common mistake:** Using `=` instead of `==` for comparison. `x = 5` assigns the value 5 to x. `x == 5` asks "is x equal to 5?" and returns `TRUE` or `FALSE`.

### Logical Operators

Combine multiple conditions using logical operators:

```r
x <- 15

x > 10 & x < 20    # TRUE  (AND: both conditions must be true)
x < 5  | x > 10    # TRUE  (OR: at least one condition must be true)
!TRUE               # FALSE (NOT: reverses the logical value)
!(x > 10)           # FALSE
```

**Vectorised logical operations:**

```r
ages <- c(12, 25, 17, 34, 8, 19)

ages >= 18                          # FALSE TRUE FALSE TRUE FALSE TRUE
ages >= 18 & ages < 30              # FALSE TRUE FALSE FALSE FALSE TRUE
!(ages >= 18)                       # TRUE FALSE TRUE FALSE TRUE FALSE
```

### Logical Values in Calculations

`TRUE` is treated as `1` and `FALSE` as `0` in arithmetic contexts. This is extremely useful for counting:

```r
passed <- c(TRUE, FALSE, TRUE, TRUE, FALSE, TRUE)

sum(passed)           # 4  (count of TRUE values)
mean(passed)          # 0.667  (proportion of TRUE values)
sum(!passed)          # 2  (count of FALSE values)
```

**Research example:** What proportion of participants had cholesterol above 200?

```r
cholesterol <- c(185, 210, 172, 240, 195, 225, 168, 198)

high_chol        <- cholesterol > 200
sum(high_chol)         # 3   (number of participants)
mean(high_chol)        # 0.375  (37.5% of participants)
```

### Checking and Converting Logical Data

```r
is.logical(TRUE)     # TRUE
is.logical(1)        # FALSE  (1 is numeric, not logical)

as.logical(1)        # TRUE
as.logical(0)        # FALSE
as.logical("TRUE")   # TRUE
as.logical("FALSE")  # FALSE
as.logical("yes")    # NA  — R only recognises "TRUE"/"FALSE", not "yes"/"no"
```

### `NA` — Missing Values

`NA` is R's symbol for a **missing value**. It is not the same as zero, an empty string, or `FALSE` — it means the value is unknown or absent.

```r
x <- NA
is.na(x)        # TRUE
x == NA         # NA  (never use == to test for NA — always use is.na())

# NA propagates through calculations
5 + NA          # NA
mean(c(1, 2, NA, 4))           # NA
mean(c(1, 2, NA, 4), na.rm = TRUE)  # 2.333...
```

## 6. Factor Data

### What is a Factor?

A **factor** is R's data type for **categorical variables** — variables that take a fixed set of possible values, called **levels**. Examples include:

- Treatment group: `"control"`, `"low_dose"`, `"high_dose"`
- Education level: `"primary"`, `"secondary"`, `"tertiary"`
- Blood type: `"A"`, `"B"`, `"AB"`, `"O"`
- Survey rating: `"strongly disagree"` through `"strongly agree"`

Factors look like character strings on the surface, but R stores them internally as integers with a lookup table of labels. This makes them memory-efficient and statistically meaningful.

### Why Factors Matter

If you use plain character strings for categorical data, R treats each value independently with no concept of grouping or order. Factors tell R: *"these values belong to a fixed set of categories"*, which matters for:

- **Statistical modelling** — regression and ANOVA treat factors correctly as categorical predictors
- **Plotting** — ggplot2 uses factor levels to control legend order and axis labels
- **Preventing errors** — factors reject values outside their defined levels

### Creating Factors

```r
# Basic factor from a character vector
treatment <- factor(c("control", "low_dose", "high_dose",
                      "control", "high_dose", "low_dose"))

treatment
# [1] control   low_dose  high_dose control   high_dose low_dose
# Levels: control high_dose low_dose

# The levels are listed at the bottom — in alphabetical order by default
levels(treatment)
# [1] "control"   "high_dose" "low_dose"

nlevels(treatment)
# [1] 3
```

### Controlling Level Order

By default, levels are sorted alphabetically. For analysis and plotting, you often need a specific order — especially for ordinal categories (those with a meaningful sequence).

```r
# Set a custom level order
treatment <- factor(
  c("control", "low_dose", "high_dose", "control", "high_dose"),
  levels = c("control", "low_dose", "high_dose")
)

levels(treatment)
# [1] "control"   "low_dose"  "high_dose"
```

**Research example:** Likert scale responses need meaningful order:

```r
responses <- c("Agree", "Strongly Agree", "Neutral", "Disagree",
               "Agree", "Strongly Disagree", "Neutral", "Agree")

responses_f <- factor(responses,
  levels = c("Strongly Disagree", "Disagree", "Neutral",
             "Agree", "Strongly Agree")
)

table(responses_f)
# responses_f
# Strongly Disagree          Disagree           Neutral
#                 1                 1                 2
#             Agree    Strongly Agree
#                 3                 1
# Note: categories appear in the correct logical order, not alphabetically
```

### Ordered Factors

When categories have a natural rank ordering (like pain severity or education level), create an **ordered factor** by setting `ordered = TRUE`.

```r
pain_level <- factor(
  c("mild", "severe", "moderate", "mild", "severe"),
  levels  = c("mild", "moderate", "severe"),
  ordered = TRUE
)

pain_level
# [1] mild     severe   moderate mild     severe
# Levels: mild < moderate < severe

# Ordered factors support comparisons
pain_level[1] < pain_level[2]   # TRUE  (mild < severe)
pain_level[3] > pain_level[1]   # TRUE  (moderate > mild)
```

### Working with Factors

```r
treatment <- factor(c("control", "low_dose", "high_dose",
                      "control", "high_dose", "low_dose"),
                    levels = c("control", "low_dose", "high_dose"))

# Frequency table — respects level order
table(treatment)
# treatment
#   control  low_dose high_dose
#         2         2         2

# Check the internal integer codes
as.integer(treatment)
# [1] 1 2 3 1 3 2

# Rename levels
levels(treatment) <- c("Control", "Low Dose", "High Dose")
treatment
# [1] Control   Low Dose  High Dose Control   High Dose Low Dose
# Levels: Control Low Dose High Dose

# Add a new level (before adding new data)
levels(treatment) <- c(levels(treatment), "Very High Dose")

# Drop unused levels
treatment_sub   <- treatment[treatment != "Very High Dose"]
treatment_clean <- droplevels(treatment_sub)
levels(treatment_clean)
# [1] "Control"   "Low Dose"  "High Dose"
```

### Converting Between Factors and Other Types

```r
f <- factor(c("low", "medium", "high", "low"))

# Factor to character
as.character(f)
# [1] "low"    "medium" "high"   "low"

# Factor to integer (returns the level codes, not the labels)
as.integer(f)
# [1] 1 3 2 1

# Character to factor
x <- c("yes", "no", "yes", "yes", "no")
x_factor <- as.factor(x)
levels(x_factor)
# [1] "no"  "yes"

# CAUTION: Converting a numeric factor to numeric
num_factor <- factor(c("10", "20", "30"))
as.numeric(num_factor)          # WRONG: 1 2 3  (level codes, not values!)
as.numeric(as.character(num_factor))  # CORRECT: 10 20 30
```

> **Critical warning:** Never convert a numeric factor directly with `as.numeric()` — you'll get the integer level codes, not the original numbers. Always go through character first: `as.numeric(as.character(factor_variable))`.

## 7. Type Coercion — What Happens When Types Mix

R will sometimes automatically convert one type to another to make an operation work. This is called **implicit coercion**, and it follows a strict hierarchy:

```
logical  ->  integer  ->  numeric  ->  character
(least flexible)                      (most flexible)
```

When types are mixed, R converts everything to the most flexible type in the mix:

```r
# Logical + Numeric -> Numeric
c(TRUE, FALSE, 1, 2)
# [1] 1 0 1 2   (TRUE becomes 1, FALSE becomes 0)

# Numeric + Character -> Character
c(1, 2, "three")
# [1] "1"     "2"     "three"   (numbers silently become strings)

# Logical + Character -> Character
c(TRUE, "hello")
# [1] "TRUE"  "hello"
```

This can cause silent bugs. For example:

```r
data <- c(10, 20, "N/A", 40)
mean(data)   # Error — data is now character, not numeric
```

Always verify your types after loading data, especially from external files:

```r
class(data)
str(data)
```

## 9. Putting It All Together — A Mini Case Study

**Scenario:** You receive a small dataset from a colleague. Before any analysis, you need to inspect, clean, and correctly type each variable.

```r
# Step 1: The raw data arrives as a data frame with mixed types
raw_data <- data.frame(
  patient_id  = c("P01","P02","P03","P04","P05","P06"),
  age         = c("45", "38", "52", "29", "61", "34"),   # stored as text!
  group       = c("A","B","A","B","A","B"),
  score       = c(8.2, 7.5, 9.1, 6.8, 8.9, 7.2),
  passed      = c("TRUE","FALSE","TRUE","FALSE","TRUE","TRUE"),
  severity    = c("mild","severe","moderate","mild","severe","moderate"),
  stringsAsFactors = FALSE
)

# Step 2: Inspect types
str(raw_data)
# 'data.frame': 6 obs. of 6 variables:
#  $ patient_id: chr  "P01" "P02" ...
#  $ age       : chr  "45" "38" ...   <- PROBLEM: age is character
#  $ group     : chr  "A" "B" ...
#  $ score     : num  8.2 7.5 9.1 6.8 8.9 7.2
#  $ passed    : chr  "TRUE" "FALSE" ...  <- PROBLEM: passed is character
#  $ severity  : chr  "mild" "severe" ...

# Step 3: Fix types
clean_data <- raw_data

# Convert age to numeric
clean_data$age <- as.numeric(clean_data$age)

# Convert passed to logical
clean_data$passed <- as.logical(clean_data$passed)

# Convert group to factor (nominal — no order)
clean_data$group <- factor(clean_data$group, levels = c("A", "B"))

# Convert severity to ordered factor
clean_data$severity <- factor(clean_data$severity,
  levels  = c("mild", "moderate", "severe"),
  ordered = TRUE
)

# Step 4: Verify the result
str(clean_data)
# 'data.frame': 6 obs. of 6 variables:
#  $ patient_id: chr  "P01" "P02" ...
#  $ age       : num  45 38 52 29 61 34
#  $ group     : Factor w/ 2 levels "A","B": 1 2 1 2 1 2
#  $ score     : num  8.2 7.5 9.1 6.8 8.9 7.2
#  $ passed    : logi  TRUE FALSE TRUE FALSE TRUE TRUE
#  $ severity  : Ord.factor w/ 3 levels "mild"<"moderate"<"severe": 1 3 2 1 3 2

# Step 5: Now analysis works correctly
mean(clean_data$age)                          # 43.17
sum(clean_data$passed)                        # 4 patients passed
table(clean_data$severity)                    # frequency by severity (in order)
clean_data$severity[1] < clean_data$severity[2]  # TRUE (mild < severe)
```

## Module Summary

In this module, you learned:

- R **syntax** follows strict rules: use `<-` for assignment, `#` for comments, and clear naming conventions
- R is **case-sensitive**: `Age` and `age` are different objects
- The four core data types are **numeric** (quantities), **character** (text), **logical** (TRUE/FALSE), and **factor** (categories)
- Each type has dedicated `is.*()` and `as.*()` functions for checking and converting
- **Factors** store categorical data efficiently and preserve category order in analysis and plots
- **Ordered factors** support comparisons (`<`, `>`) between category levels
- R uses **implicit coercion** when types mix — always check types when loading external data

1. Create three variables: a numeric storing your age, a character storing your field of study, and a logical indicating whether you have used R before. Print each one and check its type with `class()`.

2. Given `x <- c("10", "20", "30", "40")`, check its class. Then convert it to numeric, calculate the mean, and verify your result.

3. A survey collected pain ratings: `c("low", "high", "medium", "low", "high", "medium", "high")`. Create an ordered factor with the correct level sequence (`low < medium < high`). Use `table()` to count each level and verify that `"low" < "high"` evaluates to `TRUE`.

4. Given `scores <- c(55, 72, NA, 88, 61, NA, 90)`, count how many values are missing using `is.na()` and `sum()`. Then calculate the mean of the non-missing values.

5. Create a character vector of five species names. Use `toupper()`, `nchar()`, and `grepl()` to: (a) convert all names to uppercase, (b) find which names have more than 8 characters, and (c) identify which names contain the letter "a".

6. **Challenge:** A colleague gives you a vector `temperatures <- c("22.5", "19.8", "N/A", "24.1", "N/A", "21.3")` where missing values are coded as `"N/A"`. Write code to: (a) replace `"N/A"` with `NA`, (b) convert to numeric, and (c) calculate the mean ignoring missing values.
