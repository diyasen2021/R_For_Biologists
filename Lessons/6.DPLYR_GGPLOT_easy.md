## 1. Understanding Packages in R

### What is a Package?

R comes with a set of built-in functions — things like `mean()`, `sum()`, and `data.frame()`. These are part of **base R** and are always available the moment you open R.

A **package** is a collection of additional functions, datasets, and documentation that someone has written and shared with the R community. Packages extend R's capabilities far beyond what base R provides. There are packages for statistics, machine learning, bioinformatics, finance, text analysis, mapping, and much more.

Think of it like a smartphone: the phone comes with a calculator and a calendar built in (base R), but you download apps for navigation, music, and banking (packages).

### Where Do Packages Come From?

Most R packages are hosted on **CRAN** (the Comprehensive R Archive Network) — a central, curated repository of thousands of reviewed and tested packages. As of 2024, CRAN hosts over 20,000 packages.

Other sources include:

- **Bioconductor** — specialised packages for bioinformatics and genomics
- **GitHub** — development versions of packages, not yet on CRAN
- **R-universe** — an emerging alternative package repository

### The Tidyverse

`dplyr` and `ggplot2` are both part of a larger ecosystem called the **tidyverse** — a collection of R packages designed around a shared philosophy of tidy, readable, consistent data analysis. All tidyverse packages work well together and follow the same design principles.

The core tidyverse packages include:

| Package | Purpose |
|---------|---------|
| `dplyr` | Data manipulation (filter, select, summarise, join) |
| `ggplot2` | Data visualisation |
| `tidyr` | Reshaping and tidying data |
| `readr` | Fast reading of CSV and text files |
| `stringr` | Working with text strings |
| `purrr` | Functional programming with lists |
| `tibble` | A modern reimagining of data frames |

You can install all core tidyverse packages at once with `install.packages("tidyverse")`.

## 2. Installing and Loading Packages

### Installing a Package

You only need to install a package **once** per computer. Installation downloads the package files from CRAN and stores them on your machine.

```r
# Install a single package
install.packages("dplyr")
install.packages("ggplot2")

# Install both at once
install.packages(c("dplyr", "ggplot2"))

# Install the entire tidyverse (recommended)
install.packages("tidyverse")
```

> **Note:** When prompted about installing from source or whether to update other packages, it is generally safe to select "CRAN" or type `n` (no) when starting out.

### Loading a Package

Installing downloads the package — but to actually **use** it, you must load it into your current R session using `library()`. You need to do this **every time you start a new R session**.

```r
# Load packages at the top of every script
library(dplyr)
library(ggplot2)
```

### Installing vs. Loading — The Key Distinction

```
install.packages("dplyr")   ->  Downloads dplyr to your computer.
                                Do this ONCE.

library(dplyr)              ->  Makes dplyr functions available.
                                Do this at the start of EVERY session.
```

A common analogy: installing a package is like buying a book and putting it on your shelf. Loading it with `library()` is like taking it off the shelf and opening it to use.

### Checking What's Installed

```r
# Check if a specific package is installed
"dplyr" %in% rownames(installed.packages())   # TRUE or FALSE

# List all functions in a loaded package
ls("package:dplyr")

# Get help for any function
?filter
?ggplot
```

## 3. Introduction to dplyr

### What is dplyr?

`dplyr` is a package for **data manipulation**. It provides a small set of clean, readable functions — called **verbs** — that each do one clear thing to a data frame. The genius of dplyr is that these verbs are consistent, composable, and read almost like plain English.

The five core dplyr verbs are:

| Verb | What it does |
|------|-------------|
| `filter()` | Keep rows that meet a condition |
| `select()` | Keep or drop specific columns |
| `mutate()` | Create or modify columns |
| `arrange()` | Sort rows by one or more columns |
| `summarise()` | Collapse rows into summary statistics |

Each verb takes a data frame as its first argument and returns a data frame — making them easy to chain together.

### The Dataset We'll Use

Throughout this section we'll work with a patient health records dataset.

```r
library(dplyr)

health <- data.frame(
  id          = paste0("P", sprintf("%02d", 1:12)),
  age         = c(45, 38, 52, 29, 61, 34, 47, 55, 42, 31, 58, 36),
  gender      = c("F","M","F","M","F","M","F","M","F","M","F","M"),
  diet        = c("Low-Carb","Mediterranean","Low-Carb","Vegan",
                  "Mediterranean","Vegan","Low-Carb","Mediterranean",
                  "Vegan","Low-Carb","Mediterranean","Vegan"),
  bmi         = c(31.2, 27.4, 33.8, 26.1, 29.5, 28.0,
                  34.2, 30.1, 27.8, 32.5, 28.9, 25.6),
  cholesterol = c(210, 185, 240, 172, 195, 180,
                  225, 198, 175, 215, 192, 168),
  bp_systolic = c(128, 118, 142, 112, 135, 120,
                  138, 130, 115, 125, 122, 110),
  completed   = c(TRUE,TRUE,TRUE,FALSE,TRUE,TRUE,
                  TRUE,FALSE,TRUE,TRUE,TRUE,TRUE),
  stringsAsFactors = FALSE
)
```

### `filter()` — Selecting Rows by Condition

`filter()` keeps only the rows where a condition is `TRUE`. It replaces the base R `df[df$column == value, ]` pattern with cleaner syntax.

```r
# Patients over 40
filter(health, age > 40)

# Female patients only
filter(health, gender == "F")

# Patients who completed the study AND have high cholesterol
filter(health, completed == TRUE & cholesterol > 200)

# Patients on Low-Carb or Vegan diets
filter(health, diet %in% c("Low-Carb", "Vegan"))

# Multiple arguments are treated as AND conditions
filter(health, age >= 30, age <= 50)
```

### `select()` — Choosing Columns

`select()` lets you pick which columns to keep, drop, or reorder — without needing column numbers.

```r
# Keep specific columns
select(health, id, age, bmi)

# Drop a column using minus
select(health, -completed)

# Drop multiple columns
select(health, -c(completed, bp_systolic))

# Select a consecutive range
select(health, id:diet)

# Put gender first, then everything else
select(health, gender, everything())
```

`select()` also accepts helper functions for pattern-based selection:

```r
select(health, starts_with("b"))   # bmi, bp_systolic
select(health, ends_with("r"))     # gender
select(health, contains("e"))      # age, gender, diet, completed
```

### `mutate()` — Creating and Modifying Columns

`mutate()` adds new columns or transforms existing ones, returning the full data frame with your additions.

```r
# Classify BMI using case_when()
health <- mutate(health,
  bmi_category = case_when(
    bmi < 25             ~ "Normal",
    bmi >= 25 & bmi < 30 ~ "Overweight",
    bmi >= 30            ~ "Obese"
  )
)

# Add a derived clinical measure
health <- mutate(health,
  pulse_pressure = bp_systolic - 80    # simplified estimate
)

# Convert cholesterol from mg/dL to mmol/L
health <- mutate(health,
  cholesterol_mmol = round(cholesterol / 38.67, 2)
)
```

> **Note:** `case_when()` is a powerful dplyr helper that works like a multi-condition `ifelse()`. It evaluates conditions top-to-bottom and assigns the value from the first matching condition.

### `arrange()` — Sorting Rows

`arrange()` sorts a data frame by one or more columns. Wrap a column name in `desc()` to sort descending.

```r
# Sort by age, youngest first (ascending is default)
arrange(health, age)

# Sort by cholesterol, highest first
arrange(health, desc(cholesterol))

# Sort by diet group, then by BMI within each group
arrange(health, diet, bmi)

# Diet ascending, age descending within each group
arrange(health, diet, desc(age))
```

### `summarise()` — Computing Summary Statistics

`summarise()` collapses a data frame into a single row of summary statistics.

```r
summarise(health,
  n          = n(),
  mean_age   = mean(age),
  mean_bmi   = mean(bmi),
  mean_chol  = mean(cholesterol),
  max_bp     = max(bp_systolic)
)
#   n mean_age mean_bmi mean_chol max_bp
# 1 12    43.25    29.59     196.3    142
```

### `group_by()` + `summarise()` — Grouped Summaries

`group_by()` splits a data frame into invisible groups. Any subsequent `summarise()` operates on each group separately — this is one of the most useful patterns in all of data analysis.

```r
health |>
  group_by(diet) |>
  summarise(
    n         = n(),
    mean_age  = round(mean(age), 1),
    mean_bmi  = round(mean(bmi), 2),
    mean_chol = round(mean(cholesterol), 1)
  )

# A tibble: 3 x 5
#   diet              n mean_age mean_bmi mean_chol
# 1 Low-Carb          4     41.5    32.93     217.5
# 2 Mediterranean     4     47.0    28.97     192.5
# 3 Vegan             4     34.5    26.87     173.8

# Count completions by gender
health |>
  group_by(gender, completed) |>
  summarise(count = n(), .groups = "drop")
```

## 4. The Pipe Operator

### What is the Pipe?

When you need to perform several operations in sequence — filter, then select, then arrange — you would normally have to nest function calls inside each other, or create a series of intermediate objects. Both approaches get cluttered fast.

The **pipe operator** `|>` solves this by passing the output of one step directly as the input of the next, reading naturally from left to right like a set of instructions.

```r
# Without pipe — deeply nested and hard to follow
arrange(
  select(
    filter(health, age > 40),
    id, age, bmi, cholesterol
  ),
  desc(cholesterol)
)

# With pipe — reads as a clear sequence of steps
health |>
  filter(age > 40) |>
  select(id, age, bmi, cholesterol) |>
  arrange(desc(cholesterol))

# Read aloud as:
# "Take health, THEN keep rows where age > 40,
#  THEN select those four columns,
#  THEN sort by cholesterol descending."
```

> **Keyboard shortcut:** In RStudio, press `Ctrl + Shift + M` (Windows/Linux) or `Cmd + Shift + M` (Mac) to insert the pipe.

### A Complete dplyr Pipeline

```r
# Research question:
# Among completed participants with BP above 120,
# what is the average BMI and cholesterol by diet group?

results <- health |>
  filter(completed == TRUE, bp_systolic > 120) |>
  group_by(diet) |>
  summarise(
    n        = n(),
    avg_bmi  = round(mean(bmi), 2),
    avg_chol = round(mean(cholesterol), 1)
  ) |>
  arrange(avg_chol)

results
```

## 5. Introduction to ggplot2

### What is ggplot2?

`ggplot2` is R's premier data visualisation package. It is built on the **Grammar of Graphics** — a framework that describes every plot as a combination of layered components. Rather than memorising separate commands for bar charts, scatter plots, and histograms, you learn one consistent system that produces all of them.

### The Core Components

Every ggplot2 plot is built from these layers, added with `+`:

| Component | Description | Example |
|-----------|-------------|---------|
| **Data** | The data frame to plot | `data = health` |
| **Aesthetics** (`aes`) | Map variables to visual properties | `x = age, y = bmi, colour = gender` |
| **Geometry** (`geom_*`) | The type of visual mark | `geom_point()`, `geom_bar()` |
| **Labels** | Titles and axis labels | `labs(title = "...")` |
| **Scales** | Axis ranges, colour palettes | `scale_colour_manual()` |
| **Facets** | Split into sub-panels | `facet_wrap(~ diet)` |
| **Theme** | Overall visual style | `theme_minimal()` |

The basic template for every ggplot2 plot:

```r
ggplot(data = <DATA>, aes(x = <VAR>, y = <VAR>, ...)) +
  geom_*() +
  labs(...) +
  theme_*()
```

### Scatter Plot — Two Continuous Variables

```r
library(ggplot2)

# Basic scatter plot
ggplot(data = health, aes(x = age, y = bmi)) +
  geom_point()

# Colour by gender, size by cholesterol
ggplot(data = health, aes(x = age, y = bmi,
                          colour = gender, size = cholesterol)) +
  geom_point(alpha = 0.7) +
  labs(
    title    = "BMI vs Age by Gender",
    subtitle = "Point size proportional to cholesterol level",
    x        = "Age (years)",
    y        = "BMI (kg/m2)",
    colour   = "Gender",
    size     = "Cholesterol (mg/dL)"
  ) +
  theme_minimal()
```

> **Tip:** `alpha` sets transparency (0 = invisible, 1 = fully opaque). Use it whenever points overlap to reveal the density of your data.

### Histogram — Distribution of One Variable

```r
# Basic BMI distribution
ggplot(data = health, aes(x = bmi)) +
  geom_histogram(binwidth = 2, fill = "steelblue", colour = "white") +
  labs(
    title = "Distribution of BMI Scores",
    x     = "BMI (kg/m2)",
    y     = "Count"
  ) +
  theme_classic()

# Coloured by diet group
ggplot(data = health, aes(x = bmi, fill = diet)) +
  geom_histogram(binwidth = 2, colour = "white",
                 alpha = 0.7, position = "identity") +
  labs(
    title = "BMI Distribution by Diet Group",
    x     = "BMI (kg/m2)",
    y     = "Count",
    fill  = "Diet"
  ) +
  theme_minimal()
```

### Bar Chart — Comparing Groups

```r
# Count of participants per diet group (raw data)
ggplot(data = health, aes(x = diet)) +
  geom_bar(fill = "coral") +
  labs(title = "Participants per Diet Group", x = "Diet", y = "Count") +
  theme_minimal()

# Mean cholesterol per group (pre-summarised data)
chol_summary <- health |>
  group_by(diet) |>
  summarise(mean_chol = mean(cholesterol))

ggplot(data = chol_summary, aes(x = diet, y = mean_chol, fill = diet)) +
  geom_col() +
  geom_text(aes(label = round(mean_chol, 1)),
            vjust = -0.5, size = 4) +
  labs(
    title = "Average Cholesterol by Diet Group",
    x     = "Diet",
    y     = "Mean Cholesterol (mg/dL)"
  ) +
  theme_minimal() +
  theme(legend.position = "none")
```

> **`geom_bar()` vs `geom_col()`:** Use `geom_bar()` when ggplot2 should count rows for you. Use `geom_col()` when you already have the values you want to plot (e.g., from a summarised data frame).

### Box Plot — Distributions and Outliers

```r
ggplot(data = health, aes(x = diet, y = bmi, fill = diet)) +
  geom_boxplot(alpha = 0.7) +
  geom_jitter(width = 0.15, alpha = 0.6, size = 2) +
  labs(
    title = "BMI Distribution by Diet Group",
    x     = "Diet Group",
    y     = "BMI (kg/m2)"
  ) +
  theme_minimal() +
  theme(legend.position = "none")
```

> **Reading a box plot:** The middle line is the median. The box spans the interquartile range (25th to 75th percentile). Whiskers extend to 1.5x the IQR. Points beyond the whiskers are potential outliers. Adding `geom_jitter()` on top shows every individual data point.

### Line Chart — Trends Over Time

```r
# Simulate weekly blood pressure readings by diet group
weekly_bp <- data.frame(
  week          = rep(1:8, 3),
  bp            = c(130,128,126,125,122,120,119,118,   # Low-Carb
                    128,127,125,123,121,120,119,117,   # Mediterranean
                    132,129,127,124,122,121,120,119),  # Vegan
  diet          = rep(c("Low-Carb","Mediterranean","Vegan"), each = 8)
)

ggplot(data = weekly_bp, aes(x = week, y = bp,
                              colour = diet, group = diet)) +
  geom_line(linewidth = 1.2) +
  geom_point(size = 3) +
  labs(
    title  = "Weekly Average Systolic BP by Diet Group",
    x      = "Week",
    y      = "Systolic BP (mmHg)",
    colour = "Diet"
  ) +
  scale_x_continuous(breaks = 1:8) +
  theme_minimal()
```

### Faceting — Small Multiples

Faceting splits one plot into multiple panels — one per category. It is one of ggplot2's most powerful tools for comparative visualisation.

```r
# One panel per diet group
ggplot(data = health, aes(x = age, y = cholesterol, colour = gender)) +
  geom_point(size = 3, alpha = 0.8) +
  facet_wrap(~ diet) +
  labs(
    title  = "Age vs Cholesterol by Diet Group",
    x      = "Age (years)",
    y      = "Cholesterol (mg/dL)",
    colour = "Gender"
  ) +
  theme_minimal()

# Two-dimensional faceting: rows by gender, columns by diet
ggplot(data = health, aes(x = age, y = bmi)) +
  geom_point(size = 2.5) +
  facet_grid(gender ~ diet) +
  theme_bw()
```

### Themes and Colour Customisation

```r
# Built-in themes
theme_minimal()    # Clean, light, no grey background
theme_classic()    # Traditional axes, no gridlines
theme_bw()         # Black and white with grid
theme_dark()       # Dark panel background

# Manual colour palettes
ggplot(data = health, aes(x = diet, y = bmi, fill = diet)) +
  geom_boxplot() +
  scale_fill_manual(values = c(
    "Low-Carb"      = "#E76F51",
    "Mediterranean" = "#2A9D8F",
    "Vegan"         = "#457B9D"
  )) +
  labs(title = "BMI by Diet Group") +
  theme_minimal()

# Fine-tuning specific theme elements
ggplot(data = health, aes(x = age, y = bmi)) +
  geom_point(colour = "steelblue", size = 3) +
  labs(title = "Age vs BMI", x = "Age", y = "BMI") +
  theme_minimal() +
  theme(
    plot.title       = element_text(size = 16, face = "bold", hjust = 0.5),
    axis.title       = element_text(size = 12),
    axis.text        = element_text(size = 10),
    panel.grid.minor = element_blank()
  )
```

### Saving Plots with `ggsave()`

```r
# Save the most recently displayed plot
ggsave("my_plot.png", width = 8, height = 5, dpi = 300)

# Save a named plot object to a specific format
my_plot <- ggplot(health, aes(x = age, y = bmi)) + geom_point()
ggsave("age_bmi.pdf", plot = my_plot, width = 7, height = 5)

# Supported formats: "png", "pdf", "svg", "jpeg", "tiff"
```

## 6. Putting It All Together — A Mini Case Study

**Scenario:** You are preparing a summary report on the health study. You'll use `dplyr` to clean and summarise the data, then `ggplot2` to produce publication-ready visualisations.

```r
library(dplyr)
library(ggplot2)

# ── Step 1: Clean and enrich the data ────────────────────────────────────────

health_clean <- health |>
  filter(completed == TRUE) |>
  mutate(
    bmi_category = case_when(
      bmi < 25             ~ "Normal",
      bmi >= 25 & bmi < 30 ~ "Overweight",
      bmi >= 30            ~ "Obese"
    ),
    bmi_category = factor(bmi_category,
                          levels = c("Normal", "Overweight", "Obese"))
  )

# ── Step 2: Summary table by diet and gender ─────────────────────────────────

diet_summary <- health_clean |>
  group_by(diet, gender) |>
  summarise(
    n         = n(),
    mean_bmi  = round(mean(bmi), 2),
    mean_chol = round(mean(cholesterol), 1),
    mean_bp   = round(mean(bp_systolic), 1),
    .groups   = "drop"
  )

print(diet_summary)

# ── Step 3: Plot 1 — BMI by diet group, split by gender ──────────────────────

plot1 <- ggplot(health_clean, aes(x = diet, y = bmi, fill = gender)) +
  geom_boxplot(alpha = 0.7, position = position_dodge(0.8)) +
  labs(
    title    = "BMI Distribution by Diet Group and Gender",
    subtitle = "Completed participants only",
    x        = "Diet Group",
    y        = "BMI (kg/m2)",
    fill     = "Gender"
  ) +
  scale_fill_manual(values = c("F" = "#E76F51", "M" = "#457B9D")) +
  theme_minimal() +
  theme(plot.title = element_text(face = "bold", hjust = 0.5))

print(plot1)
ggsave("plot1_bmi_by_diet.png", plot = plot1, width = 8, height = 5, dpi = 300)

# ── Step 4: Plot 2 — Cholesterol vs BP, coloured by BMI category ─────────────

plot2 <- ggplot(health_clean,
                aes(x = bp_systolic, y = cholesterol,
                    colour = bmi_category)) +
  geom_point(size = 3.5, alpha = 0.85) +
  facet_wrap(~ diet) +
  labs(
    title  = "Cholesterol vs Systolic BP by Diet Group",
    x      = "Systolic BP (mmHg)",
    y      = "Cholesterol (mg/dL)",
    colour = "BMI Category"
  ) +
  theme_bw() +
  theme(
    strip.text      = element_text(face = "bold"),
    legend.position = "bottom"
  )

print(plot2)
ggsave("plot2_chol_bp_facet.png", plot = plot2,
       width = 10, height = 5, dpi = 300)
```

## Module Summary

In this module, you learned:

- A **package** extends base R with extra functions. Install once with `install.packages()`; load every session with `library()`.
- **dplyr** provides five core verbs — `filter()`, `select()`, `mutate()`, `arrange()`, `summarise()` — plus `group_by()` for grouped operations.
- The **pipe operator** `|>` chains steps in a left-to-right, readable flow.
- **ggplot2** builds plots layer by layer using a consistent grammar: `ggplot()` + `aes()` + `geom_*()` + optional scales, labels, facets, and themes.
- `ggsave()` exports any plot to PNG, PDF, SVG, and other formats.

### Key Functions Reference

| Package | Function | Purpose |
|---------|----------|---------|
| base R | `install.packages()` | Download and install a package |
