## 1. What is dplyr?

**dplyr** is the tidyverse package for transforming and summarising data.  
It gives you a small set of functions — called **verbs** — that cover the vast majority of data manipulation tasks.

### Why dplyr over base R?

- Code reads like plain English — `filter()`, `select()`, `arrange()` say exactly what they do
- Works seamlessly with the **pipe operator** — chain steps together without nested brackets
- Consistent behaviour — every verb takes a data frame in and returns a data frame out
- Fast — optimised for performance on large datasets

### The four verb groups

dplyr verbs are organised by **what they operate on**:

| Group | Verbs | What they do |
|---|---|---|
| **Rows** | `filter()`, `arrange()`, `slice()` | Keep, remove, or reorder rows |
| **Columns** | `select()`, `rename()`, `mutate()`, `relocate()` | Keep, rename, create, or reorder columns |
| **Groups** | `group_by()`, `ungroup()`, `summarise()` | Aggregate and summarise by category |
| **Tables** | `left_join()`, `inner_join()`, `bind_rows()` | Combine multiple data frames |

> We will cover Rows, Columns, and Groups in this tutorial.  
> Table verbs (joins) will be covered in a later session.

## 3. The Pipe Operator

### The problem without pipes

Suppose you want to:
1. Filter to only United Airlines (`"UA"`) flights
2. Then keep only the `flight`, `origin`, `dest`, and `dep_delay` columns
3. Then sort by departure delay descending

Without a pipe, you have two options — both are awkward:

```r
# Option A — nested functions (read inside-out, hard to follow)
arrange(
  select(
    filter(flights, carrier == "UA"),
    flight, origin, dest, dep_delay
  ),
  desc(dep_delay)
)

# Option B — intermediate objects (clutters your environment)
step1 <- filter(flights, carrier == "UA")
step2 <- select(step1, flight, origin, dest, dep_delay)
step3 <- arrange(step2, desc(dep_delay))
```

### The pipe solves this

The pipe operator **`|>`** takes the result on the left and passes it as the first argument to the function on the right.

```
data |> function()   is the same as   function(data)
```

```r
# The same three steps — now readable top to bottom
flights |>
  filter(carrier == "UA") |>
  select(flight, origin, dest, dep_delay) |>
  arrange(desc(dep_delay))
```

- Read it as: *"take flights, then filter to United, then select those columns, then sort"*
- Each line does one thing
- Easy to add, remove, or reorder steps
- The result is only stored if you assign it: `result <- flights |> filter(...)`

### The two pipe operators you will see

```r
# |>  — the native R pipe (R 4.1+), no packages needed
flights |> filter(carrier == "UA")

# %>%  — the magrittr pipe from the tidyverse (older code uses this)
flights %>% filter(carrier == "UA")
```

> For new code, use `|>`.  
> You will see `%>%` in older tutorials and Stack Overflow answers — they behave the same for our purposes.

## 4. Row Verbs — filter() and arrange()

Row verbs work on **rows** — they never add, remove, or rename columns.

---

### `filter()` — keep rows that match a condition

```r
# Single condition — flights to Miami only
flights |> filter(dest == "MIA")
```

```
# A tibble: 11,728 × 19
    year month   day dep_time sched_dep_time dep_delay arr_time sched_arr_time
   <int> <int> <int>    <int>          <int>     <dbl>    <int>          <int>
 1  2013     1     1      542            540         2      923            850
 2  2013     1     1      613            610         3      925            921
 3  2013     1     1      706            700         6     1007            955
...
```

```r
# Multiple conditions — AND (both must be true)
# Flights to Miami in January
flights |> filter(dest == "MIA", month == 1)

# Multiple conditions — OR (either can be true)
# Flights to Miami OR Los Angeles
flights |> filter(dest == "MIA" | dest == "LAX")

# Equivalent shorthand for OR on the same column
flights |> filter(dest %in% c("MIA", "LAX", "SFO"))

# NOT — exclude an airport
flights |> filter(origin != "LGA")

# Numeric comparisons — badly delayed flights
flights |> filter(dep_delay >= 120)         # 2+ hours late departing
flights |> filter(arr_delay > 0)            # arrived late at all
flights |> filter(dep_delay < 0)            # left early

# Combining conditions — late departure from JFK
flights |> filter(origin == "JFK", dep_delay > 60)

# Flights that made up time in the air (arrived less late than they departed)
flights |> filter(dep_delay > 0, arr_delay < dep_delay)

# Filter on missing values — flights with no recorded departure time (cancelled)
flights |> filter(is.na(dep_time))          # 8,255 cancelled flights
flights |> filter(!is.na(dep_time))         # only flights that actually departed
```

### Comparison operators — quick reference

| Operator | Meaning | Example |
|---|---|---|
| `==` | equal to | `carrier == "UA"` |
| `!=` | not equal to | `origin != "LGA"` |
| `>` | greater than | `dep_delay > 60` |
| `>=` | greater than or equal | `arr_delay >= 0` |
| `<` | less than | `dep_delay < 0` |
| `<=` | less than or equal | `air_time <= 60` |
| `%in%` | is one of these values | `dest %in% c("MIA", "LAX")` |
| `!` | NOT | `!is.na(dep_time)` |
| `&` or `,` | AND | `month == 12, day == 25` |
| `\|` | OR | `month == 11 \| month == 12` |

### `arrange()` — sort rows

```r
# Sort by departure delay ascending — most early first
flights |> arrange(dep_delay)

# Sort by departure delay descending — worst delays first
flights |> arrange(desc(dep_delay))
```

```

   <int> <int> <int>    <int>          <int>     <dbl> <chr>   <chr>

 2  2013     6    15     1432           1935      1137 MQ      CMH
 3  2013     1    10     1121           1635      1126 MQ      OME

```

```r
# Sort by multiple columns — by month, then day, then departure time
flights |> arrange(month, day, dep_time)

# Sort by distance descending — longest flights first
flights |> arrange(desc(distance))

# NAs always sort to the bottom in arrange()
flights |> arrange(dep_delay)   # cancelled flights (NA dep_delay) appear last
```

---

### Combining filter and arrange

```r
# Worst departure delays from JFK specifically
flights |>
  filter(origin == "JFK") |>
  arrange(desc(dep_delay))

# Flights that departed early AND arrived late — interesting anomaly
flights |>
  filter(dep_delay < 0, arr_delay > 0) |>
  arrange(desc(arr_delay))
```

## 5. Column Verbs — select(), rename(), mutate()

Column verbs work on **columns** — they never add or remove rows.

---

### `select()` — keep or drop columns

```r
# Keep specific columns by name
flights |> select(year, month, day, carrier, dest, dep_delay, arr_delay)

# Keep a range of adjacent columns
flights |> select(year:dep_time)

# Drop columns with -
flights |> select(-time_hour, -minute, -hour)

# Drop multiple columns
flights |> select(-c(year, month, day, time_hour))

# Reorder — put carrier and dest first, keep everything else
flights |> select(carrier, dest, everything())
```

### Handy helper functions inside `select()`

```r
# Columns whose name starts with a string
flights |> select(starts_with("dep"))     # dep_time, dep_delay

# Columns whose name ends with a string
flights |> select(ends_with("delay"))     # dep_delay, arr_delay

# Columns whose name contains a string
flights |> select(contains("time"))       # dep_time, arr_time, air_time, time_hour

# Select only numeric columns
flights |> select(where(is.numeric))

# Select only character columns
flights |> select(where(is.character))    # carrier, tailnum, origin, dest
```

---

### `rename()` — rename columns

```r
# rename(new_name = old_name)
flights |> rename(
  departure_delay = dep_delay,
  arrival_delay   = arr_delay,
  tail_number     = tailnum
)
```

> `rename()` only changes names — it never drops columns.  
> `select()` can also rename while selecting: `select(departure_delay = dep_delay, carrier)`

---

### `mutate()` — add or change columns

`mutate()` adds new columns or overwrites existing ones.  
New columns appear at the right side of the tibble.

```r
# Calculate speed in miles per hour
flights |>
  mutate(speed_mph = distance / (air_time / 60)) |>
  select(flight, carrier, distance, air_time, speed_mph)
```

```
# A tibble: 336,776 × 5
   flight carrier distance air_time speed_mph
    <int> <chr>      <dbl>    <dbl>     <dbl>
 1   1545 UA          1400      227      370.
 2   1714 UA          1400      227      370.
 3   1141 AA          1089      160      408.
...
```

```r
# Add multiple columns at once — and reference a new column immediately
flights |>
  mutate(
    speed_mph   = distance / (air_time / 60),
    gain        = dep_delay - arr_delay,        # time made up in the air
    gain_per_hr = gain / (air_time / 60)
  ) |>
  select(flight, carrier, dep_delay, arr_delay, gain, gain_per_hr)

# Convert dep_delay from minutes to hours
flights |>
  mutate(dep_delay_hrs = round(dep_delay / 60, 2))

# Conditional column — flag long-haul flights
flights |>
  mutate(
    flight_type = ifelse(distance >= 2000, "Long-haul", "Short-haul")
  ) |>
  select(flight, dest, distance, flight_type)

# Multiple categories with case_when()
flights |>
  mutate(
    delay_category = case_when(
      dep_delay < 0   ~ "Early",
      dep_delay == 0  ~ "On time",
      dep_delay <= 30 ~ "Minor delay",
      dep_delay <= 120 ~ "Significant delay",
      dep_delay > 120  ~ "Severe delay"
    )
  ) |>
  select(flight, carrier, dep_delay, delay_category)
```

```
 5    461 DL             -6 Early
...
```

## 6. Group Verbs — group_by() and summarise()

This is where dplyr becomes genuinely powerful.  
Instead of operating on the whole dataset, you can split it into groups and run calculations **per group**.

---

### `summarise()` — collapse rows into summary statistics

Without grouping, `summarise()` collapses the whole dataset into one row:

```r
flights |>
  summarise(
    total_flights  = n(),
    mean_dep_delay = mean(dep_delay, na.rm = TRUE),
    mean_arr_delay = mean(arr_delay, na.rm = TRUE),
    pct_cancelled  = round(mean(is.na(dep_time)) * 100, 2)
  )
```

```
# A tibble: 1 × 4
  total_flights mean_dep_delay mean_arr_delay pct_cancelled
          <int>          <dbl>          <dbl>         <dbl>
1        336776           12.6           6.90          2.45
```

### Common summary functions

| Function | What it calculates |
|---|---|
| `n()` | count of rows |
| `n_distinct(x)` | count of unique values |
| `sum(x)` | total |
| `mean(x)` | average |
| `median(x)` | median |
| `min(x)` | minimum |
| `max(x)` | maximum |
| `sd(x)` | standard deviation |
| `first(x)` | first value in the group |
| `last(x)` | last value in the group |

> Always add `na.rm = TRUE` inside numeric functions — `flights` has NAs in delay columns for cancelled flights.  
> Example: `mean(dep_delay, na.rm = TRUE)`

---

### `group_by()` — split into groups before summarising

`group_by()` does not change the data — it attaches grouping information.  
The magic happens when you chain it with `summarise()`.

```r
# Average departure delay per airline carrier
flights |>
  group_by(carrier) |>
  summarise(
    flights        = n(),
    mean_dep_delay = round(mean(dep_delay, na.rm = TRUE), 1),
    mean_arr_delay = round(mean(arr_delay, na.rm = TRUE), 1)
  ) |>
  arrange(desc(mean_dep_delay)) |>
  ungroup()
```

### `group_by()` + `mutate()` — add a group calculation back to the original rows

Sometimes you want the group summary **alongside the original rows**, not collapsed into one row per group.  
Use `mutate()` instead of `summarise()` after `group_by()`.

```r
# Add each carrier's average delay alongside every individual flight
# Useful for comparing each flight to its airline's typical performance
flights |>
  group_by(carrier) |>
  mutate(
    carrier_avg_delay = mean(dep_delay, na.rm = TRUE),
    delay_vs_avg      = dep_delay - carrier_avg_delay
  ) |>
  select(flight, carrier, dep_delay, carrier_avg_delay, delay_vs_avg) |>
  ungroup()
```


### `ungroup()` — always clean up after grouping

After `group_by()`, the tibble stays grouped until you explicitly remove the grouping.  
Forgetting to `ungroup()` can cause unexpected results in later steps.

```r
# Safe pattern — always ungroup when done
flights |>
  group_by(carrier) |>
  summarise(mean_delay = mean(dep_delay, na.rm = TRUE)) |>
  ungroup()

# Check whether a tibble is currently grouped
flights_grouped <- flights |> group_by(carrier)
is.grouped_df(flights_grouped)     # TRUE

flights_ungrouped <- flights_grouped |> ungroup()
is.grouped_df(flights_ungrouped)   # FALSE
```

> **Rule of thumb:** if you assign a grouped result to a variable, always `ungroup()` at the end.

## 8. dplyr Cheat Sheet

```r
library(dplyr)
library(nycflights13)

# ── Row verbs ─────────────────────────────────────────────────────────
flights |> filter(carrier == "UA")                  # keep matching rows
flights |> filter(dest %in% c("MIA", "LAX"))        # keep rows in a list
flights |> filter(!is.na(dep_time))                 # remove NA rows
flights |> arrange(dep_delay)                       # sort ascending
flights |> arrange(desc(dep_delay))                 # sort descending
flights |> arrange(month, day, desc(dep_delay))     # sort by multiple columns
flights |> slice_max(dep_delay, n = 10)             # top 10 rows by dep_delay
flights |> slice_min(dep_delay, n = 10)             # bottom 10 rows by dep_delay

# ── Column verbs ──────────────────────────────────────────────────────
flights |> select(carrier, dest, dep_delay)         # keep columns
flights |> select(-time_hour, -minute)              # drop columns
flights |> select(ends_with("delay"))               # columns ending in "delay"
flights |> select(where(is.numeric))                # numeric columns only
flights |> rename(departure_delay = dep_delay)      # rename a column
flights |> mutate(speed = distance / (air_time/60)) # add a column
flights |> mutate(                                  # conditional column
  delay_cat = case_when(
    dep_delay <  0  ~ "Early",
    dep_delay == 0  ~ "On time",
    dep_delay <= 30 ~ "Minor",
    TRUE            ~ "Severe"
  )
)

# ── Group verbs ───────────────────────────────────────────────────────
flights |>
  group_by(carrier) |>
  summarise(
    count      = n(),
    mean_delay = mean(dep_delay, na.rm = TRUE),
    max_delay  = max(dep_delay, na.rm = TRUE)
  ) |>
  ungroup()

# group_by + mutate (keeps all rows, adds group calculation alongside them)
flights |>
  group_by(carrier) |>
  mutate(carrier_avg = mean(dep_delay, na.rm = TRUE)) |>
  ungroup()

# ── Full pipeline reminder ────────────────────────────────────────────
result <- flights |>
  filter(...) |>       # step 1 — rows
  select(...) |>       # step 2 — columns
  mutate(...) |>       # step 3 — new columns
  group_by(...) |>     # step 4 — grouping
  summarise(...) |>    # step 5 — aggregate
  arrange(...) |>      # step 6 — sort
  ungroup()            # always finish with ungroup() if you used group_by
```
