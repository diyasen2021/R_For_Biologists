## 1. Understanding Lists and Their Structure

### What is a List?

In the previous module, you learned that a **vector** holds multiple values — but all of the same type. What if you want to store a mix of numbers, text, and even other vectors together? That's where **lists** come in.

A **list** in R is an ordered collection of objects that can be of **different types and different lengths**. Each item in a list is called an **element**, and each element can itself be a vector, another list, a data frame, a function, or any R object.

Think of a list like a filing cabinet: each drawer can hold something completely different — one drawer might hold numbers, another text, another a whole spreadsheet.

### Vectors vs. Lists — A Quick Comparison

| Feature | Vector | List |
|---------|--------|------|
| Data types | All the same | Can be mixed |
| Element types | Single values | Any R object |
| Use case | Homogeneous data | Heterogeneous / complex data |
| Access syntax | `x[1]` | `x[[1]]` or `x$name` |

```r
# A vector — all numeric, same type
my_vector <- c(1, 2, 3, 4)

# A list — mixed types
my_list <- list(42, "hello", TRUE, c(1, 2, 3))
```

## 2. Creating and Accessing Elements in a List

### Creating a List with `list()`

Use the `list()` function to create a list. Elements can be named or unnamed.

```r
# Unnamed list
patient_info <- list("Alice", 34, TRUE, c(120, 118, 122))

# Named list (recommended — much easier to work with)
patient_info <- list(
  name    = "Alice",
  age     = 34,
  diabetic = FALSE,
  bp_readings = c(120, 118, 122)
)
```

### Inspecting a List

```r
# Check the structure — your best friend for lists
str(patient_info)
# List of 4
#  $ name       : chr "Alice"
#  $ age        : num 34
#  $ diabetic   : logi FALSE
#  $ bp_readings: num [1:3] 120 118 122

# How many elements?
length(patient_info)
# [1] 4

# Check the names
names(patient_info)
# [1] "name" "age" "diabetic" "bp_readings"
```

> **Tip:** `str()` is incredibly useful. It shows you the structure, types, and a preview of every element in one clean output. Always use it when exploring an unfamiliar list.

### Accessing List Elements

R gives you three ways to access list elements. Understanding when to use each is important.

#### Method 1: Double Square Brackets `[[]]` — Returns the Element Itself

```r
patient_info[[1]]       # By position: "Alice"
patient_info[["age"]]   # By name: 34
patient_info[["bp_readings"]]  # Returns the vector: 120 118 122
```

#### Method 2: The `$` Operator — Access by Name (Most Common)

```r
patient_info$name         # "Alice"
patient_info$age          # 34
patient_info$bp_readings  # 120 118 122

# You can even go deeper — access elements within the nested vector
patient_info$bp_readings[2]   # Second reading: 118
```

#### Method 3: Single Square Brackets `[]` — Returns a Sub-List

```r
# Returns a LIST containing just the first element (not the element itself)
patient_info[1]
# $name
# [1] "Alice"

# Useful for extracting multiple elements as a new list
patient_info[c("name", "age")]
```

#### The Key Difference: `[[]]` vs `[]`

```r
# [[]] extracts the element — you get the actual value
patient_info[["age"]]        # 34  (a number)
class(patient_info[["age"]]) # "numeric"

# [] subsets the list — you still get a list back
patient_info["age"]          # $age: 34  (still a list)
class(patient_info["age"])   # "list"
```

> **Rule of thumb:** Use `$` or `[[]]` when you want to work with the value. Use `[]` when you want to keep the result as a list.

### Modifying List Elements

```r
# Update an existing element
patient_info$age <- 35

# Add a new element
patient_info$hospital <- "City General"

# Remove an element by assigning NULL
patient_info$diabetic <- NULL

str(patient_info)
# List of 4
#  $ name       : chr "Alice"
#  $ age        : num 35
#  $ bp_readings: num [1:3] 120 118 122
#  $ hospital   : chr "City General"
```

### Nested Lists

Lists can contain other lists, making them suitable for complex, hierarchical data.

```r
study <- list(
  study_id   = "EXP-001",
  lead_researcher = "Dr. Smith",
  participants = list(
    list(id = 1, name = "Alice", score = 88),
    list(id = 2, name = "Bob",   score = 74),
    list(id = 3, name = "Carol", score = 91)
  )
)

# Access nested elements
study$participants[[2]]$name    # "Bob"
study$participants[[3]]$score   # 91
```

### Converting a List to a Vector

If a list contains only elements of the same type, you can flatten it into a vector with `unlist()`:

```r
measurements <- list(12.4, 15.1, 9.8, 14.0)
unlist(measurements)
# [1] 12.4 15.1  9.8 14.0
```

## 3. Introduction to Data Frames

### What is a Data Frame?

A **data frame** is the most important data structure for data analysis in R. It is a **table** — like a spreadsheet — where:

- Each **column** is a variable (and a vector of a single type)
- Each **row** is an observation
- All columns have the **same number of rows**
- Different columns **can** hold different types

If you've worked with Excel, CSV files, or SPSS, a data frame is the R equivalent.

```
| patient_id | age | treatment | response |
|------------|-----|-----------|----------|
| P001       |  45 | Drug A    |   TRUE   |
| P002       |  38 | Placebo   |   FALSE  |
| P003       |  52 | Drug A    |   TRUE   |
```

### Data Frames vs. Lists vs. Vectors

| Feature | Vector | List | Data Frame |
|---------|--------|------|------------|
| Dimensions | 1D | 1D (but nested) | 2D (rows & columns) |
| Mixed types | ✗ | ✓ | ✓ (by column) |
| Best for | Single variable | Complex/mixed objects | Tabular datasets |
| Typical use | Calculations | Storing model outputs | Data analysis |

## 4. Creating and Manipulating Data Frames

### Creating a Data Frame with `data.frame()`

```r
# Each argument becomes a column
clinical_data <- data.frame(
  patient_id  = c("P001", "P002", "P003", "P004", "P005"),
  age         = c(45, 38, 52, 29, 61),
  treatment   = c("Drug A", "Placebo", "Drug A", "Placebo", "Drug A"),
  response    = c(TRUE, FALSE, TRUE, FALSE, TRUE),
  score       = c(8.2, 5.1, 9.0, 4.7, 8.8),
  stringsAsFactors = FALSE   # Keep text as characters, not factors
)
```

### Inspecting a Data Frame

These are the first commands you should run whenever you load a new dataset:

```r
# Dimensions: rows x columns
nrow(clinical_data)   # 5
ncol(clinical_data)   # 5
dim(clinical_data)    # 5 5

# Column names
names(clinical_data)
# [1] "patient_id" "age" "treatment" "response" "score"

# Overview of structure and types
str(clinical_data)
# 'data.frame': 5 obs. of 5 variables:
#  $ patient_id: chr  "P001" "P002" "P003" "P004" ...
#  $ age       : num  45 38 52 29 61
#  $ treatment : chr  "Drug A" "Placebo" "Drug A" "Placebo" ...
#  $ response  : logi  TRUE FALSE TRUE FALSE TRUE
#  $ score     : num  8.2 5.1 9 4.7 8.8

# Quick statistical summary of all columns
summary(clinical_data)
#  patient_id             age         treatment           response
#  Length:5          Min.   :29.0   Length:5          Mode :logical
#  Class :character  1st Qu.:38.0   Class :character  FALSE:2
#  Mode  :character  Median :45.0   Mode  :character  TRUE :3
#                    Mean   :45.0
#                    3rd Qu.:52.0
#                    Max.   :61.0
#     score
#  Min.   :4.700
#  1st Qu.:5.100
#  Median :8.200
#  Mean   :7.160
#  3rd Qu.:8.800
#  Max.   :9.000

# Preview first and last rows
head(clinical_data)     # First 6 rows (default)
head(clinical_data, 3)  # First 3 rows
tail(clinical_data, 2)  # Last 2 rows
```

### Accessing Columns

```r
# Using $ — most common
clinical_data$age
# [1] 45 38 52 29 61

clinical_data$treatment
# [1] "Drug A"  "Placebo" "Drug A"  "Placebo" "Drug A"

# Using [[]] — same result
clinical_data[["score"]]
# [1] 8.2 5.1 9.0 4.7 8.8
```

### Accessing Rows and Cells

Data frames use **two-dimensional indexing**: `df[row, column]`

```r
clinical_data[1, ]        # Entire first row
clinical_data[, 2]        # Entire second column (age)
clinical_data[1, 2]       # Row 1, Column 2: 45
clinical_data[2, "score"] # Row 2, score column: 5.1

# Multiple rows
clinical_data[1:3, ]      # First 3 rows
clinical_data[c(1,3,5), ] # Rows 1, 3, and 5
```

### Filtering Rows (Subsetting)

This is one of the most frequent operations in data analysis — selecting only the rows that meet a condition.

```r
# Patients who responded (response == TRUE)
responders <- clinical_data[clinical_data$response == TRUE, ]
responders
#   patient_id age treatment response score
# 1       P001  45    Drug A     TRUE   8.2
# 3       P003  52    Drug A     TRUE   9.0
# 5       P005  61    Drug A     TRUE   8.8

# Patients older than 40
older_patients <- clinical_data[clinical_data$age > 40, ]

# Patients on Drug A with a score above 8
drug_a_high <- clinical_data[
  clinical_data$treatment == "Drug A" & clinical_data$score > 8,
]
drug_a_high
#   patient_id age treatment response score
# 1       P001  45    Drug A     TRUE   8.2
# 3       P003  52    Drug A     TRUE   9.0
# 5       P005  61    Drug A     TRUE   8.8
```

> **Tip:** The `subset()` function offers a cleaner syntax for filtering:
> ```r
> subset(clinical_data, treatment == "Drug A" & score > 8)
> ```

### Adding New Columns

```r
# Add a column that classifies score as "High" or "Low"
clinical_data$score_group <- ifelse(clinical_data$score >= 8, "High", "Low")

# Add a column with normalized scores (0–1 scale)
clinical_data$score_norm <- (clinical_data$score - min(clinical_data$score)) /
                            (max(clinical_data$score) - min(clinical_data$score))

clinical_data
#   patient_id age treatment response score score_group score_norm
# 1       P001  45    Drug A     TRUE   8.2        High      0.816
# 2       P002  38   Placebo    FALSE   5.1         Low      0.093
# 3       P003  52    Drug A     TRUE   9.0        High      1.000
# 4       P004  29   Placebo    FALSE   4.7         Low      0.000
# 5       P005  61    Drug A     TRUE   8.8        High      0.953
```

### Removing Columns

```r
# Remove a column by setting it to NULL
clinical_data$score_norm <- NULL

# Or create a new data frame without certain columns
clinical_small <- clinical_data[, c("patient_id", "age", "score")]
```

### Adding New Rows with `rbind()`

`rbind()` (row bind) appends rows to a data frame.

```r
# Create a new patient record
new_patient <- data.frame(
  patient_id  = "P006",
  age         = 47,
  treatment   = "Placebo",
  response    = FALSE,
  score       = 6.3,
  score_group = "Low",
  stringsAsFactors = FALSE
)

# Append to the existing data frame
clinical_data <- rbind(clinical_data, new_patient)
nrow(clinical_data)  # Now 6 rows
```

### Combining Data Frames Side-by-Side with `cbind()`

`cbind()` (column bind) adds columns to a data frame.

```r
# A second data frame with additional measurements
extra_data <- data.frame(
  follow_up_score = c(7.9, 5.5, 9.1, 4.5, 8.5, 6.0)
)

clinical_data <- cbind(clinical_data, extra_data)
```

> ⚠️ **Warning:** When using `cbind()`, make sure the row order matches exactly between the two data frames. There is no automatic alignment.

### Sorting a Data Frame

Use `order()` to sort a data frame by one or more columns.

```r
# Sort by age (ascending)
clinical_data[order(clinical_data$age), ]

# Sort by score (descending)
clinical_data[order(-clinical_data$score), ]

# Sort by treatment, then by score descending
clinical_data[order(clinical_data$treatment, -clinical_data$score), ]
```

### Renaming Columns

```r
# Rename all columns at once
names(clinical_data) <- c("id", "age", "tx", "responded", "score", "group")

# Rename a specific column by position
names(clinical_data)[3] <- "treatment"

# Rename by matching name (safer)
names(clinical_data)[names(clinical_data) == "tx"] <- "treatment"
```

### Checking for Missing Values in a Data Frame

```r
# Introduce some NAs for demonstration
clinical_data$score[2] <- NA

# Check for any NAs in the entire data frame
anyNA(clinical_data)        # TRUE

# Which cells are NA?
is.na(clinical_data)

# Count NAs per column
colSums(is.na(clinical_data))
# patient_id  age  treatment  responded  score  group
#          0    0          0          0      1      0

# Remove all rows with any NA values
clean_data <- na.omit(clinical_data)
```

## 5. Loading Real Data into a Data Frame

In practice, you'll rarely type data in by hand. Most real-world data comes from files. The most common format is CSV (comma-separated values).

### Reading a CSV File

```r
# Read a CSV file from your computer
my_data <- read.csv("path/to/your/file.csv", stringsAsFactors = FALSE)

# Read with a semicolon separator (common in European data)
my_data <- read.csv2("path/to/your/file.csv")

# Always inspect immediately after loading
dim(my_data)
str(my_data)
head(my_data)
summary(my_data)
```

### Writing a Data Frame to a CSV File

```r
# Save your data frame to a CSV file
write.csv(clinical_data, "clinical_results.csv", row.names = FALSE)
```

> **Note:** Setting `row.names = FALSE` prevents R from writing row numbers as a column in your CSV.

## 6. Putting It All Together — A Mini Case Study

**Scenario:** You are managing data from a small ecology study. You have survey records of bird sightings across three sites over two seasons.

```r
# Step 1: Build the data frame
birds <- data.frame(
  site       = c("Forest", "Wetland", "Forest", "Grassland", "Wetland", "Grassland"),
  season     = c("Spring", "Spring", "Autumn", "Spring", "Autumn", "Autumn"),
  species    = c("Robin", "Heron", "Robin", "Skylark", "Heron", "Skylark"),
  count      = c(14, 3, 9, 7, 5, 4),
  avg_weight = c(18.2, 1450.0, 17.8, 37.5, 1480.0, 36.8),
  stringsAsFactors = FALSE
)

# Step 2: Inspect the data
str(birds)
summary(birds)

# Step 3: Filter — only Forest sightings
forest_birds <- subset(birds, site == "Forest")
forest_birds
#      site season species count avg_weight
# 1  Forest Spring   Robin    14       18.2
# 3  Forest Autumn   Robin     9       17.8

# Step 4: Add a column — total estimated biomass per sighting
birds$total_biomass <- birds$count * birds$avg_weight
birds
#        site season  species count avg_weight total_biomass
# 1    Forest Spring    Robin    14       18.2         254.8
# 2   Wetland Spring    Heron     3     1450.0        4350.0
# 3    Forest Autumn    Robin     9       17.8         160.2
# 4 Grassland Spring  Skylark     7       37.5         262.5
# 5   Wetland Autumn    Heron     5     1480.0        7400.0
# 6 Grassland Autumn  Skylark     4       36.8         147.2

# Step 5: Which site had the highest total biomass?
birds[which.max(birds$total_biomass), ]
#      site season species count avg_weight total_biomass
# 5 Wetland Autumn   Heron     5     1480.0          7400

# Step 6: Sort by count (descending)
birds[order(-birds$count), ]

# Step 7: Summary statistics on count
mean(birds$count)    # 7
sum(birds$count)     # 42 total sightings
```

## Module Summary

In this module, you learned:

- A **list** is a flexible container that can hold elements of any type and length — ideal for complex or mixed data.
- List elements are accessed with `$`, `[[]]`, or `[]`, each returning a different result.
- A **data frame** is R's primary structure for tabular data — like a spreadsheet with typed columns.
- You can create, inspect, filter, modify, and sort data frames using base R tools.
- Real-world data is typically loaded with `read.csv()` and saved with `write.csv()`.

### Key Functions Reference

| Function | Purpose |
|----------|---------|
| `list()` | Create a list |
| `str()` | Inspect structure of any object |
| `names()` | Get or set names |
| `length()` | Number of elements |
| `data.frame()` | Create a data frame |
| `nrow()`, `ncol()`, `dim()` | Dimensions of a data frame |
| `head()`, `tail()` | Preview rows |
| `summary()` | Statistical summary |
| `subset()` | Filter rows with a condition |
| `rbind()` | Add rows |
| `cbind()` | Add columns |
| `order()` | Sort by column(s) |
| `is.na()`, `na.omit()` | Handle missing values |
| `read.csv()` | Load data from a CSV file |
| `write.csv()` | Save data frame to a CSV file |
