## 1. What is dplyr?

**dplyr** is the tidyverse package for transforming and summarising data.  
It gives you a small set of functions — called **verbs** — that cover the vast majority of data manipulation tasks.

### Why dplyr over base R?

- Code reads like plain English — `filter()`, `select()`, `arrange()` say exactly what they do
- Works seamlessly with the **pipe operator** — chain steps together without nested brackets
- Consistent behaviour — every verb takes a data frame in and returns a data frame out
- Fast — optimised for performance on large datasets

### The four verb groups

dplyr verbs are organised by **what they operate on**:

| Group | Verbs | What they do |
|---|---|---|
| **Rows** | `filter()`, `arrange()`, `slice()` | Keep, remove, or reorder rows |
| **Columns** | `select()`, `rename()`, `mutate()`, `relocate()` | Keep, rename, create, or reorder columns |
| **Groups** | `group_by()`, `ungroup()`, `summarise()` | Aggregate and summarise by category |
| **Tables** | `left_join()`, `inner_join()`, `bind_rows()` | Combine multiple data frames |

> We will cover Rows, Columns, and Groups in this tutorial.  
> Table verbs (joins) will be covered in a later session.

## 3. The Pipe Operator

### The problem without pipes

Suppose you want to:
1. Filter to only Laptop sales
2. Then keep only the rep and units columns
3. Then sort by units descending

Without a pipe, you have two options — both are awkward:

```r
# Option A — nested functions (read inside-out, hard to follow)
arrange(
  select(
    filter(sales, product == "Laptop"),
    rep, units
  ),
  desc(units)
)

# Option B — intermediate objects (clutters your environment)
step1 <- filter(sales, product == "Laptop")
step2 <- select(step1, rep, units)
step3 <- arrange(step2, desc(units))
```

### The pipe solves this

The pipe operator **`|>`** takes the result on the left and passes it as the first argument to the function on the right.

```
data |> function()   is the same as   function(data)
```

```r
# The same three steps — now readable top to bottom
sales |>
  filter(product == "Laptop") |>
  select(rep, units) |>
  arrange(desc(units))
```

- Read it as: *"take sales, then filter, then select, then arrange"*
- Each line does one thing
- Easy to add, remove, or reorder steps
- The result is only stored if you assign it: `result <- sales |> filter(...)`

### The two pipe operators you will see

```r
# |>  — the native R pipe (R 4.1+), no packages needed
sales |> filter(product == "Laptop")

# %>%  — the magrittr pipe from the tidyverse (older code uses this)
sales %>% filter(product == "Laptop")
```

> For new code, use `|>`.  
> You will see `%>%` in older tutorials and Stack Overflow answers — they behave the same for our purposes.

## 4. Row Verbs — filter() and arrange()

Row verbs work on **rows** — they never add, remove, or rename columns.

---

### `filter()` — keep rows that match a condition

```r
# Single condition — one product only
sales |> filter(product == "Laptop")
```

```
# A tibble: 5 × 9
  order_id rep   region product units unit_price discount date       returned
     <int> <chr> <chr>  <chr>   <dbl>      <dbl>    <dbl> <date>     <lgl>
1        1 Alice North  Laptop      3       1200     0.10 2024-01-05 FALSE
2        4 Carol East   Laptop      4       1200     0.15 2024-02-14 TRUE
...
```

```r
# Multiple conditions — AND (both must be true)
sales |> filter(product == "Laptop", units > 3)

# Multiple conditions — OR (either can be true)
sales |> filter(product == "Laptop" | product == "Phone")

# Equivalent shorthand for OR on the same column
sales |> filter(product %in% c("Laptop", "Phone"))

# NOT — exclude a value
sales |> filter(product != "Tablet")

# Numeric comparisons
sales |> filter(units >= 4)
sales |> filter(unit_price > 500 & discount > 0.05)

# Filter out returned orders
sales |> filter(returned == FALSE)
# or equivalently:
sales |> filter(!returned)

# Date filtering
sales |> filter(date >= as.Date("2024-04-01"))

# Filter on missing values
sales |> filter(is.na(rep))       # rows where rep is NA
sales |> filter(!is.na(rep))      # rows where rep is NOT NA
```

### Comparison operators — quick reference

| Operator | Meaning | Example |
|---|---|---|
| `==` | equal to | `product == "Laptop"` |
| `!=` | not equal to | `product != "Tablet"` |
| `>` | greater than | `units > 3` |
| `>=` | greater than or equal | `units >= 4` |
| `<` | less than | `discount < 0.10` |
| `<=` | less than or equal | `unit_price <= 800` |
| `%in%` | is one of these values | `product %in% c("Laptop", "Phone")` |
| `!` | NOT | `!returned` |
| `&` or `,` | AND | `units > 3 & region == "North"` |
| `\|` | OR | `product == "Laptop" \| product == "Phone"` |

---

### `arrange()` — sort rows

```r
# Sort by units ascending (smallest first — default)
sales |> arrange(units)

# Sort by units descending (largest first)
sales |> arrange(desc(units))

# Sort by multiple columns — first by region, then by units descending
sales |> arrange(region, desc(units))

# Sort by date
sales |> arrange(date)
```

```
# A tibble: 15 × 9
  order_id rep   region product units unit_price ...
     <int> <chr> <chr>  <chr>   <dbl>      <dbl>
1        7 Carol East   Tablet      1        450    ← smallest units first
2        3 Alice North  Tablet      2        450
3       10 Carol East   Laptop      2       1200
...
```

---

### Combining filter and arrange

```r
# Find the top orders by units — Laptops only, sorted largest first
sales |>
  filter(product == "Laptop") |>
  arrange(desc(units))
```

## 5. Column Verbs — select(), rename(), mutate()

Column verbs work on **columns** — they never add or remove rows.

---

### `select()` — keep or drop columns

```r
# Keep specific columns by name
sales |> select(rep, product, units)

# Keep a range of columns
sales |> select(order_id:units)

# Drop a column with -
sales |> select(-returned)
sales |> select(-c(discount, returned))

# Reorder columns — put rep and product first, keep everything else
sales |> select(rep, product, everything())
```

### Handy helper functions inside `select()`

```r
# Columns whose name starts with a string
sales |> select(starts_with("unit"))       # unit_price, units

# Columns whose name ends with a string
sales |> select(ends_with("_price"))       # unit_price

# Columns whose name contains a string
sales |> select(contains("_"))             # order_id, unit_price

# Select by data type
sales |> select(where(is.numeric))
sales |> select(where(is.character))
```

---

### `rename()` — rename columns

```r
# rename(new_name = old_name)
sales |> rename(
  sales_rep    = rep,
  sale_date    = date,
  was_returned = returned
)
```

> `rename()` only changes names — it never drops columns.  
> `select()` can also rename while selecting: `select(sales_rep = rep, product)`

---

### `mutate()` — add or change columns

`mutate()` adds new columns or overwrites existing ones.  
The new column is always added at the right side of the tibble.

```r
# Add a revenue column
sales |>
  mutate(revenue = units * unit_price)
```

```
# A tibble: 15 × 10
  order_id rep   region product units unit_price discount date       returned revenue
     <int> <chr> <chr>  <chr>   <dbl>      <dbl>    <dbl> <date>     <lgl>    <dbl>
1        1 Alice North  Laptop      3       1200     0.10 2024-01-05 FALSE     3600
2        2 Bob   South  Phone       5        800     0.05 2024-01-12 FALSE     4000
...
```

```r
# Add multiple columns at once — and use a new column immediately
sales |>
  mutate(
    revenue          = units * unit_price,
    revenue_after_discount = revenue * (1 - discount),    # uses revenue just created
    high_value       = revenue_after_discount > 3000
  )

# Overwrite an existing column — convert discount to percentage display
sales |>
  mutate(discount = discount * 100)   # 0.10 becomes 10

# Conditional column with ifelse()
sales |>
  mutate(
    deal_size = ifelse(units >= 5, "Large", "Small")
  )

# Multiple categories with case_when() — like a chain of ifelse
sales |>
  mutate(
    deal_size = case_when(
      units >= 5  ~ "Large",
      units >= 3  ~ "Medium",
      units <  3  ~ "Small"
    )
  )
```

## 6. Group Verbs — group_by() and summarise()

This is where dplyr becomes genuinely powerful.  
Instead of operating on the whole dataset, you can split it into groups and run calculations **per group**.

---

### `summarise()` — collapse rows into summary statistics

Without grouping, `summarise()` collapses the whole dataset into one row:

```r
sales |>
  summarise(
    total_orders   = n(),
    total_units    = sum(units),
    avg_unit_price = mean(unit_price),
    total_revenue  = sum(units * unit_price)
  )
```

```
# A tibble: 1 × 4
  total_orders total_units avg_unit_price total_revenue
         <int>       <dbl>          <dbl>         <dbl>
1           15          53           817.         50650
```

### Common summary functions

| Function | What it calculates |
|---|---|
| `n()` | count of rows |
| `n_distinct(x)` | count of unique values |
| `sum(x)` | total |
| `mean(x)` | average |
| `median(x)` | median |
| `min(x)` | minimum |
| `max(x)` | maximum |
| `sd(x)` | standard deviation |
| `first(x)` | first value in the group |
| `last(x)` | last value in the group |

> Always add `na.rm = TRUE` inside numeric functions if your data has NAs:  
> `mean(units, na.rm = TRUE)`

---

### `group_by()` — split into groups before summarising

`group_by()` does not change the data — it attaches grouping information.  
The magic happens when you chain it with `summarise()`.

```r
# Total units and revenue per sales rep
sales |>
  group_by(rep) |>
  summarise(
    orders        = n(),
    total_units   = sum(units),
    total_revenue = sum(units * unit_price)
  )
```

```
# A tibble: 3 × 4
  rep   orders total_units total_revenue
  <chr>  <int>       <dbl>         <dbl>
1 Alice      6          23         22800
2 Bob        5          23         20650
3 Carol      4           9          7200
```

```r
# Group by multiple columns — rep AND product
sales |>
  group_by(rep, product) |>
  summarise(
    orders      = n(),
    total_units = sum(units),
    avg_price   = mean(unit_price)
  )
```

```
# A tibble: 9 × 5
# Groups: rep [3]
  rep   product orders total_units avg_price
  <chr> <chr>    <int>       <dbl>     <dbl>
1 Alice Laptop       3          12      1200
2 Alice Phone        3          12       800
3 Bob   Laptop       1           5      1200
...
```

```r
# Average discount by region, sorted highest to lowest
sales |>
  group_by(region) |>
  summarise(avg_discount = mean(discount)) |>
  arrange(desc(avg_discount))
```

---

### `group_by()` + `mutate()` — add a group calculation back to the original rows

Sometimes you want the group summary **alongside the original rows**, not collapsed into one row per group.  
Use `mutate()` instead of `summarise()` after `group_by()`.

```r
# Add each rep's total revenue as a new column (without collapsing rows)
sales |>
  group_by(rep) |>
  mutate(
    rep_total_revenue = sum(units * unit_price),
    rep_pct_of_total  = round((units * unit_price) / rep_total_revenue * 100, 1)
  ) |>
  select(order_id, rep, product, units, rep_total_revenue, rep_pct_of_total)
```

```
# A tibble: 15 × 6
# Groups: rep [3]
  order_id rep   product units rep_total_revenue rep_pct_of_total
     <int> <chr> <chr>   <dbl>             <dbl>            <dbl>
1        1 Alice Laptop      3             22800             15.8
2        3 Alice Tablet      2             22800              3.9
...
```

---

### `ungroup()` — always clean up after grouping

After `group_by()`, the tibble stays grouped until you explicitly remove the grouping.  
Forgetting to `ungroup()` can cause unexpected results in later steps.

```r
# Safe pattern — always ungroup when done
sales |>
  group_by(rep) |>
  summarise(total_units = sum(units)) |>
  ungroup()

# Check whether a tibble is currently grouped
sales_grouped <- sales |> group_by(rep)
is.grouped_df(sales_grouped)   # TRUE

sales_ungrouped <- sales_grouped |> ungroup()
is.grouped_df(sales_ungrouped) # FALSE
```

> **Rule of thumb:** if you assign a grouped result to a variable, always `ungroup()` at the end.

## 7. Putting It All Together

Real analysis chains multiple verbs together.  
Here are some realistic examples using the sales dataset.

---

### Example 1 — Revenue leaderboard by rep

```r
sales |>
  filter(!returned) |>                                        # exclude returns
  mutate(revenue = units * unit_price * (1 - discount)) |>   # net revenue
  group_by(rep) |>
  summarise(
    orders        = n(),
    total_revenue = round(sum(revenue), 2),
    avg_order     = round(mean(revenue), 2)
  ) |>
  arrange(desc(total_revenue)) |>
  ungroup()
```

### Example 2 — Best-selling product per region

```r
sales |>
  group_by(region, product) |>
  summarise(total_units = sum(units), .groups = "drop") |>
```

## 8. dplyr Cheat Sheet

```r
library(dplyr)

# ── Row verbs ─────────────────────────────────────────────────
filter(df, col == "value")           # keep matching rows
filter(df, col %in% c("a", "b"))     # keep rows where col is a or b
filter(df, !is.na(col))              # remove NA rows
arrange(df, col)                     # sort ascending
arrange(df, desc(col))               # sort descending
arrange(df, col1, desc(col2))        # sort by multiple columns
slice_max(df, col, n = 5)            # top 5 rows by col
slice_min(df, col, n = 5)            # bottom 5 rows by col

# ── Column verbs ──────────────────────────────────────────────
select(df, col1, col2)               # keep columns
select(df, -col1)                    # drop a column
select(df, starts_with("x"))         # keep columns starting with "x"
select(df, where(is.numeric))        # keep numeric columns
rename(df, new_name = old_name)      # rename a column
mutate(df, new_col = col1 * col2)    # add a column
mutate(df, col = ifelse(cond, a, b)) # conditional column
mutate(df, col = case_when(          # multi-condition column
  cond1 ~ "value1",
  cond2 ~ "value2",
  TRUE  ~ "other"
))

# ── Group verbs ───────────────────────────────────────────────
df |> group_by(col) |>
  summarise(
    count     = n(),
    total     = sum(x),
    average   = mean(x, na.rm = TRUE),
    minimum   = min(x),
    maximum   = max(x)
  ) |>
  ungroup()

# group_by + mutate (keeps all rows, adds group calculation)
df |>
  group_by(col) |>
  mutate(group_total = sum(x)) |>
  ungroup()

# ── Pipe reminder ─────────────────────────────────────────────
result <- df |>
  filter(...) |>      # step 1
  mutate(...) |>      # step 2
  group_by(...) |>    # step 3
  summarise(...) |>   # step 4
  arrange(...) |>     # step 5
  ungroup()           # always finish with ungroup() if you used group_by
```
