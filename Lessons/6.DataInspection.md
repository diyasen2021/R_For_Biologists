## 1. What is a Tibble?

A **tibble** is the tidyverse's modern version of a data frame.  
It stores data in the same rows-and-columns structure, but behaves more safely and prints more clearly.

### data.frame vs tibble — the key differences

| | `data.frame` | `tibble` |
|---|---|---|
| **Prints** | Dumps every row to the console | Shows first 10 rows, fits to screen width |
| **Column types shown** | No | Yes — shown under each column name |
| **Strings to factors** | Depends on R version / settings | Never — always stays as character |
| **Partial column name matching** | Yes — `df$spec` finds `species` silently | No — must use the exact name |
| **Row names** | Supported | Not used |
| **`[` subsetting** | Can silently return a vector | Always returns a tibble |
| **Created by** | `data.frame()` / `read.csv()` | `tibble()` / `read_csv()` |

### Why does partial matching matter?

```r
# data.frame — this works silently, even if you made a typo
df_base <- data.frame(species = c("Adelie", "Gentoo"), island = c("Biscoe", "Dream"))
df_base$spec      # Returns the species column — no warning!

# tibble — this fails loudly
df_tbl <- tibble(species = c("Adelie", "Gentoo"), island = c("Biscoe", "Dream"))
df_tbl$spec       # Returns NULL with a warning — protects you from silent bugs
```

> **Bottom line:** tibbles are stricter than data frames.  
> They refuse to guess what you meant, which catches errors early.

## 4. Checking Data Types

### Why types matter so much

- R performs **different operations** depending on type — you can average body mass (numeric) but not species names (character)
- A column stored as the **wrong type** will silently produce wrong results or cause errors
- **Factor columns** are especially important for biology — they encode categorical groupings like species and sex that R needs to handle correctly in models and plots

### The type system — what you will encounter

| Type | Shows as | Stores | Penguin example |
|---|---|---|---|
| `double` | `<dbl>` | Decimal numbers | `bill_length_mm`, `bill_depth_mm` |
| `integer` | `<int>` | Whole numbers | `flipper_length_mm`, `body_mass_g`, `year` |
| `character` | `<chr>` | Text strings | Names, notes, free text |
| `logical` | `<lgl>` | TRUE / FALSE | Yes/no fields |
| `Date` | `<date>` | Calendar dates | Collection date |
| `factor` | `<fct>` | Categories with levels | `species`, `island`, `sex` |

### Checking types

```r
# Option 1 — glimpse() (recommended — shows all at once)
glimpse(penguins)

# Option 2 — str() (base R equivalent, slightly more verbose)
str(penguins)
# tibble [344 × 8] (S3: tbl_df/tbl/data.frame)
#  $ species          : Factor w/ 3 levels "Adelie","Chinstrap","Gentoo": 1 1 1 1 1 ...
#  $ island           : Factor w/ 3 levels "Biscoe","Dream","Torgersen": 3 3 3 3 3 ...
#  $ bill_length_mm   : num  39.1 39.5 40.3 NA 36.7 39.3 38.9 39.2 34.1 42 ...
#  $ body_mass_g      : int  3750 3800 3250 NA 3450 3650 3625 4675 3475 4250 ...
```

## 5. The Type Coercion Problem

### What is type coercion?

**Coercion** is when R forces a value into a type it was not meant to be.  
This happens silently and is one of the most common sources of bugs in data analysis.

### The most dangerous case — numeric read as character

This happens when a numeric column contains **even one non-numeric value**.  
R falls back to reading the whole column as character — and you won't know until something breaks.

```r
# Simulate a common field data scenario:
# Someone typed "not measured" in the body mass column for one penguin
penguins_problem <- read.csv(
  text = "species,island,body_mass_g
Adelie,Torgersen,3750
Adelie,Torgersen,not measured
Adelie,Torgersen,3250
Gentoo,Biscoe,5500"
)

glimpse(penguins_problem)
# $ species     <chr> "Adelie", "Adelie", "Adelie", "Gentoo"
# $ island      <chr> "Torgersen", "Torgersen", "Torgersen", "Biscoe"
# $ body_mass_g <chr> "3750", "not measured", "3250", "5500"
#                      ↑ entire column is now character because of one text entry!
```

### Why is this dangerous?

```r
# Calculating mean body mass fails silently
mean(penguins_problem$body_mass_g)
# Warning: argument is not numeric or logical: returning NA
# [1] NA

# Comparisons fail
penguins_problem$body_mass_g > 4000
# Error: comparison is not allowed for character vectors

# Sorting gives alphabetical order — biologically meaningless
sort(penguins_problem$body_mass_g)
# [1] "3250"        "3750"        "5500"        "not measured"
#      ↑ sorted as text — "5" comes after "3" alphabetically, happens to look right here
#      but "not measured" sorts last, not as NA
```

### How to detect it

```r
# Always run glimpse() immediately after loading
glimpse(penguins_problem)   # spot <chr> where you expect <dbl>

# Or check a specific column
is.numeric(penguins_problem$body_mass_g)    # FALSE — problem confirmed
is.character(penguins_problem$body_mass_g)  # TRUE
```

### How to fix it

```r
# Step 1 — find the non-numeric values
unique(penguins_problem$body_mass_g)
# [1] "3750"         "not measured" "3250"         "5500"

# Step 2 — replace the problem text with NA
penguins_problem$body_mass_g[penguins_problem$body_mass_g == "not measured"] <- NA

# Step 3 — convert to numeric
penguins_problem$body_mass_g <- as.numeric(penguins_problem$body_mass_g)

penguins_problem$body_mass_g
# [1] 3750   NA 3250 5500

# Now calculations work correctly
mean(penguins_problem$body_mass_g, na.rm = TRUE)
# [1] 4166.667
```

### Fixing types in the penguins CSV scenario

```r
# Read the CSV version where species, island, sex came in as character
penguins_csv <- read.csv("penguins.csv")

# Convert character columns to factors
penguins_csv$species <- as.factor(penguins_csv$species)
penguins_csv$island  <- as.factor(penguins_csv$island)
penguins_csv$sex     <- as.factor(penguins_csv$sex)

# Verify
levels(penguins_csv$species)   # [1] "Adelie"    "Chinstrap" "Gentoo"
levels(penguins_csv$sex)       # [1] "female"    "male"

glimpse(penguins_csv)
# $ species <fct> Adelie, Adelie, ...   ← now a factor
# $ island  <fct> Torgersen, ...        ← now a factor
# $ sex     <fct> male, female, ...     ← now a factor
```

### The `as.*()` function family — quick reference

| Function | Converts to | Watch out for |
|---|---|---|
| `as.numeric()` | double / numeric | Non-numeric strings become `NA` with a warning |
| `as.integer()` | integer | Decimals are truncated, not rounded |
| `as.character()` | character | Always works — everything can become text |
| `as.logical()` | TRUE / FALSE | Only works on `0`/`1` or `"TRUE"`/`"FALSE"` strings |
| `as.Date()` | Date | Must supply `format =` if not `YYYY-MM-DD` |
| `as.factor()` | factor | Levels are set alphabetically by default |

> **Key rule:** If `as.numeric()` produces `NA` values that were not `NA` before,  
> there were non-numeric strings in the column. Find them with `unique()` first.

```r
# Safe pattern — check before converting
x <- c("3750", "3800", "not measured", "3250")

# Find the problem values
x[is.na(as.numeric(x))]
# [1] "not measured"   ← now you know exactly what to clean

# Fix, then convert
x[x == "not measured"] <- NA
as.numeric(x)
# [1] 3750 3800   NA 3250
```

## 6. Checking for Missing Values

### Why missing values matter — especially in field biology

- Field data almost always has missing values — equipment fails, weather intervenes, animals escape measurement
- Missing values in `penguins` are real: two penguins had no blood drawn, so sex could not be determined
- Many R functions return `NA` if the input contains `NA` — unless you tell them to ignore it
- You need to know **how many**, **which columns**, and **whether there is a pattern** (e.g. are NAs clustered in one species?)

### Counting missing values

```r
# Is a specific value missing?
is.na(penguins$sex)
# [1] FALSE FALSE FALSE  TRUE FALSE ... (TRUE where sex was not recorded)

# Total NAs in one column
sum(is.na(penguins$sex))
# [1] 11

# Total NAs across every column
colSums(is.na(penguins))
#           species            island    bill_length_mm     bill_depth_mm
#                 0                 0                 2                 2
# flipper_length_mm       body_mass_g               sex              year
#                 2                 2                11                 0
```

```r
# Total NAs in the entire dataset
sum(is.na(penguins))
# [1] 19

# Percentage missing per column — which columns have the most missingness?
round(colMeans(is.na(penguins)) * 100, 1)
#           species            island    bill_length_mm     bill_depth_mm
#               0.0               0.0               0.6               0.6
# flipper_length_mm       body_mass_g               sex              year
#               0.6               0.6               3.2               0.0
```

- `sex` has the most missingness — **3.2%** of penguins have no recorded sex
- The measurement columns (`bill_length_mm`, `bill_depth_mm`, `flipper_length_mm`, `body_mass_g`) all have exactly **2 NAs** — almost certainly the same two penguins

### Finding the rows that contain NAs

```r
# Which rows have at least one NA anywhere?
penguins[!complete.cases(penguins), ]
```

```
# A tibble: 11 × 8
   species island    bill_length_mm bill_depth_mm flipper_length_mm body_mass_g sex    year
   <fct>   <fct>              <dbl>         <dbl>             <int>       <int> <fct> <int>
 1 Adelie  Torgersen             NA            NA                NA          NA NA     2007
 2 Adelie  Torgersen             NA            NA                NA          NA NA     2007  ← same two penguins
 3 Adelie  Dream                 NA            NA                NA          NA NA     2009  ← missing everything
...
```

```r
# complete.cases() returns TRUE for rows with no NAs at all
sum(complete.cases(penguins))
# [1] 333   ← 333 penguins have a full set of measurements

# How many rows have ANY missing value?
sum(!complete.cases(penguins))
# [1] 11
```

### Visualising missing values (optional — requires naniar)

```r
# install.packages("naniar")
library(naniar)

# Quick visual heatmap of missing values
vis_miss(penguins)
# Shows sex has the most missingness, measurements are missing together

# Which combinations of columns tend to be missing at the same time?
gg_miss_upset(penguins)
# Reveals that bill, flipper, body_mass, and sex are often all missing for the same penguin
```

### What to do about missing values

This is a biological decision as much as a statistical one:

| Option | When to use it |
|---|---|
| **Leave as NA** | Functions like `mean(x, na.rm = TRUE)` handle it; missingness itself is informative |
| **Drop the rows** | `na.omit()` — only if missingness is rare and unrelated to biology |
| **Impute (fill in)** | Replace with mean, median, or a model — for more advanced analysis |
| **Flag the missingness** | Add a column `sex_missing` — keeps the information visible |

```r
# Option 1 — drop all rows with any NA
penguins_complete <- na.omit(penguins)
nrow(penguins_complete)   # [1] 333

# Option 2 — drop rows with NA in one specific column only
# e.g. keep all penguins that have body mass recorded
penguins_measured <- penguins[!is.na(penguins$body_mass_g), ]
nrow(penguins_measured)   # [1] 342

# Option 3 — flag missing sex instead of dropping
# Useful if you want to keep all 344 rows for morphology analysis
penguins$sex_missing <- is.na(penguins$sex)
table(penguins$sex_missing)
# FALSE  TRUE
#   333    11
```

> **Biological note:** the two penguins missing all measurements (rows 4 and 272)  
> are almost certainly the same individuals where blood could not be drawn.  
> Dropping them for morphological analyses is appropriate.  
> But for a study on *missingness itself*, you would want to keep them.

## 7. Checking for Duplicates

### Why duplicates matter in biological data

- Duplicate records can arise when data from multiple field seasons is merged
- A duplicated penguin row would inflate counts and bias means
- In `penguins` this is unlikely (each row is one penguin measured once),  
  but in occurrence datasets and lab records it is extremely common

### Detecting duplicates

```r
# Are there any fully duplicated rows?
anyDuplicated(penguins)
# [1] 0   ← good — no duplicate rows in the clean penguins dataset

# Confirm with duplicated()
sum(duplicated(penguins))
# [1] 0
```

The clean `penguins` dataset has no duplicates. Let's simulate what it looks like when there are:

```r
# Simulate a dataset where one penguin was accidentally entered twice
penguins_dup <- rbind(penguins, penguins[1, ])   # add row 1 again at the bottom

anyDuplicated(penguins_dup)
# [1] 345   ← row 345 is a duplicate (non-zero = yes there are duplicates)

# Which rows are duplicates?
sum(duplicated(penguins_dup))
# [1] 1

# See the duplicate row
penguins_dup[duplicated(penguins_dup), ]
# A tibble: 1 × 8
#   species island    bill_length_mm bill_depth_mm flipper_length_mm body_mass_g sex    year
#   <fct>   <fct>              <dbl>         <dbl>             <int>       <int> <fct> <int>
# 1 Adelie  Torgersen           39.1          18.7               181        3750 male   2007

# See both the original and its duplicate side by side
penguins_dup[duplicated(penguins_dup) | duplicated(penguins_dup, fromLast = TRUE), ]
# Returns rows 1 and 345 — identical in every column
```

### Checking for duplicates on a key column

In the raw penguins data, there is no unique ID column.  
In your own field data, there usually should be — a ring number, sample ID, or specimen code.

```r
# Example: check that individual IDs are unique in your own dataset
# (using a hypothetical ring_id column)

# table() quickly shows how many times each value appears
table(penguins$island)
# Biscoe Dream Torgersen
#    168    124        52

# All island values appear many times — that is expected for a grouping variable
# But if this were an ID column, every value should appear exactly once

# Check for any value appearing more than once
any(table(penguins$island) > 1)   # TRUE — expected for island (not an ID)
```

### Removing duplicates

```r
# Remove fully duplicated rows — base R (keeps first occurrence)
penguins_clean <- penguins_dup[!duplicated(penguins_dup), ]
nrow(penguins_clean)   # [1] 344   ← back to the original count
```

## 8. Inspection Cheat Sheet

Run these in order every time you load a new biological dataset:

```r
library(palmerpenguins)
library(tibble)

df <- penguins    # or: df <- read_csv("your_field_data.csv")

# ── Step 1: First look ─────────────────────────────────────────
glimpse(df)              # rows, columns, types, sample values — run this first
head(df)                 # first 6 rows — do values look biologically sensible?
tail(df)                 # last 6 rows — does the dataset end cleanly?
dim(df)                  # rows × columns
names(df)                # column names

# ── Step 2: Check types ────────────────────────────────────────
sapply(df, class)        # type of every column
# Flag: any <chr> that should be <fct>, <dbl>, or <date>?
# Common biology issues:
#   species/sex/treatment read as <chr> instead of <fct>
#   body_mass read as <chr> because of a "not measured" entry
#   collection_date read as <chr> instead of <date>

# ── Step 3: Fix wrong types ────────────────────────────────────
df$species     <- as.factor(df$species)
df$sex         <- as.factor(df$sex)
df$body_mass_g <- as.numeric(df$body_mass_g)
df$date_col    <- as.Date(df$date_col, format = "%Y-%m-%d")

# ── Step 4: Check for missing values ───────────────────────────
colSums(is.na(df))                       # NAs per column
round(colMeans(is.na(df)) * 100, 1)     # % missing per column
sum(complete.cases(df))                  # number of fully complete rows
df[!complete.cases(df), ]               # inspect the rows with NAs

# ── Step 5: Check for duplicates ───────────────────────────────
anyDuplicated(df)                        # 0 = no duplicates
sum(duplicated(df))                      # count of duplicate rows
df[duplicated(df), ]                     # see the duplicate rows
df_clean <- df[!duplicated(df), ]       # remove duplicates

# ── Step 6: Quick summary ──────────────────────────────────────
summary(df)
# For factors: shows count per level — e.g. Adelie: 152, Chinstrap: 68, Gentoo: 124
# For numerics: shows min, median, mean, max, and count of NAs
```
