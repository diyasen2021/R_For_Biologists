## 1. What is readr and Why Use It?

**readr** is part of the **tidyverse** — a collection of R packages designed to work together for data science.

### Key reasons to use `read_csv()` over base R's `read.csv()`:

- **Faster** — uses a C++ backend, noticeably quicker on large files
- **Transparent** — prints a column specification every time you load a file, so you always know what R inferred
- **Safer** — warns you when values fail to parse instead of silently changing the column type
- **Cleaner output** — returns a **tibble** instead of a data frame (prints neatly, never converts strings to factors)
- **Preserves column names** — spaces and special characters are kept as-is, nothing is renamed silently

### Installing and loading readr

```r
# Install once — choose the full tidyverse bundle or readr on its own
install.packages("tidyverse")   # full bundle
install.packages("readr")       # readr only

# Load at the start of every session
library(readr)
```

> **Note:** You only ever run `install.packages()` once. You run `library()` every time you start a new R session.

## 3. Reading CSV Files with `read_csv()`

### What happens when you call `read_csv()`?

- R opens the file and reads each row as one observation
- The first row becomes column names (by default)
- readr samples the first 1,000 rows to **guess** each column's type
- It prints a **column specification summary** — this is not an error, it is useful feedback
- The result is a **tibble**

### Basic usage

```r
employees <- read_csv("employees.csv")
```

readr immediately tells you what it found:

```
Rows: 6 Columns: 6
── Column specification ──────────────────────────────────────
Delimiter: ","
chr (3): first_name, last_name, department
dbl (2): employee_id, salary
date (1): hire_date

ℹ Use `spec()` to retrieve the full column specification.
```

### The tibble output

```r
employees
```

```
# A tibble: 6 × 6
  employee_id first_name last_name department     salary hire_date
        <dbl> <chr>      <chr>     <chr>           <dbl> <date>
1           1 Alice      Smith     Finance         72000 2019-03-15
2           2 Bob        Jones     HR              65000 2021-07-01
3           3 Carol      White     Engineering     95000 2018-11-20
4           4 David      Brown     Finance         68000 2020-05-10
5           5 Eve        Davis     Marketing       71000 2022-01-30
6           6 Frank      Miller    Engineering    102000 2017-08-14
```

- The **type label** under each column name (`<dbl>`, `<chr>`, `<date>`) tells you exactly how R stored it
- Base R data frames do not show this — you have to run `str()` to find out

### Inspecting your tibble

```r
# Dimensions
dim(employees)           # [1] 6 6

# Column names
names(employees)         # [1] "employee_id" "first_name" ...

# Types and structure

# tibble [6 × 6] (S3: tbl_df/tbl/data.frame)
#  $ employee_id: num  1 2 3 4 5 6

#  $ salary     : num  72000 65000 95000 68000 71000 102000

# Quick statistics
summary(employees)

spec(employees)
```

### Key Arguments of `read_csv()`

| Argument | Default | What it does |
|---|---|---|
| `file` | `""` | Path to the CSV file, or a URL |
| `col_names` | `TRUE` | First row used as column names; or pass a character vector |
| `col_types` | `NULL` | Explicit column types — see Section 4 |
| `col_select` | `NULL` | Read only a subset of columns by name |
| `na` | `c("", "NA")` | Strings treated as missing (NA) |
| `skip` | `0` | Lines to skip before reading |
| `n_max` | `Inf` | Maximum rows to read |
| `skip_empty_rows` | `TRUE` | Drop completely blank rows automatically |
| `show_col_types` | `TRUE` | Print column spec after loading; set `FALSE` to suppress |
| `locale` | `default_locale()` | Controls encoding, decimal marks, date formats |
| `comment` | `""` | Lines starting with this character are ignored |
| `trim_ws` | `TRUE` | Strip leading/trailing whitespace from each field |

## 4. Controlling Column Types with `col_types`

### Why specify types explicitly?

- readr guesses types by sampling the **first 1,000 rows** — this can go wrong on larger or messier files
- Specifying types makes your scripts **reproducible** — the same code always produces the same result
- If a future version of the file contains unexpected values, readr will **warn you** rather than silently importing wrong data
- When you set `col_types`, readr suppresses the column spec message (you've already declared what you expect)

---

### Option 1 — Compact string notation

One letter per column, in order:

| Letter | Type |
|---|---|
| `c` | character |
| `d` | double (numeric) |
| `i` | integer |
| `l` | logical (TRUE/FALSE) |
| `D` | date |
| `T` | datetime |
| `_` | skip this column |
| `?` | let readr guess |

```r
# i = integer, c = character, c, c, d = double, D = date
employees <- read_csv("employees.csv", col_types = "icccdd")
```

---

### Option 2 — The `cols()` function (recommended for scripts)

More readable, names each column explicitly:

```r
employees <- read_csv("employees.csv",
  col_types = cols(
    employee_id = col_integer(),
    first_name  = col_character(),
    last_name   = col_character(),
    department  = col_factor(),        # treat as categorical
    salary      = col_double(),
    hire_date   = col_date(format = "%Y-%m-%d")
  )
)
```

**Available `col_*()` functions:**

| Function | Reads as |
|---|---|
| `col_double()` | numeric with decimals |
| `col_integer()` | whole numbers |
| `col_character()` | text |
| `col_logical()` | TRUE / FALSE |
| `col_factor()` | categorical factor |
| `col_date(format=)` | date — specify format string |
| `col_datetime(format=)` | date + time |
| `col_skip()` | ignore this column |
| `col_guess()` | let readr infer (the default) |

---

### Reading only selected columns

```r
# Select by name
employees <- read_csv("employees.csv",
                      col_select = c(employee_id, first_name, salary))

# Exclude a column
employees <- read_csv("employees.csv",
                      col_select = -last_name)
```

## 5. Common Pitfalls

### 5.1 — Column Headers with Spaces

**The scenario:** Your CSV file uses headers like `First Name` and `Annual Salary`.

```r
# First, create a version of the file with spaces in headers
employees_spaces <- employees
names(employees_spaces) <- c("Employee ID", "First Name", "Last Name",
                              "Department", "Annual Salary", "Hire Date")
write_csv(employees_spaces, "employees_spaces.csv")
```

**What read.csv() does:**
- Silently renames `First Name` → `First.Name`
- No warning — you only discover this later when code breaks

**What read_csv() does:**
- Preserves the name exactly: `First Name`
- You must use **backtick notation** to access it

```r
df <- read_csv("employees_spaces.csv")

# Accessing columns with spaces — backticks are required
df$`First Name`              # Correct
df$First.Name                # Returns NULL — this column does not exist!
df$`First Name`[1]           # "Alice"

# In tidyverse pipelines
library(dplyr)
df |> select(`First Name`, `Annual Salary`)
```

**The best fix — rename immediately after import:**

```r
df <- read_csv("employees_spaces.csv")

# Option A: rename manually
names(df) <- c("employee_id", "first_name", "last_name",
               "department", "salary", "hire_date")

# Option B: use janitor for automatic snake_case conversion
# install.packages("janitor")
library(janitor)
df <- df |> clean_names()
# Result: employee_id, first_name, last_name, department, salary, hire_date
```

> **Rule of thumb:** Clean your column headers *before* importing where possible.  
> Use `snake_case` naming — no spaces, no capitals.

### 5.2 — Missing Values

**The scenario:** Missing data is encoded in different ways across systems.

```r
# Create a version of the dataset with various missing value styles
employees_missing <- data.frame(
  employee_id = 1:6,
  first_name  = c("Alice", "Bob", "Carol", "David", "Eve", "Frank"),
  department  = c("Finance", "HR", "Engineering", "Finance", NA, "Engineering"),
  salary      = c(72000, "N/A", 95000, -999, 71000, "NULL"),
  stringsAsFactors = FALSE
)
write_csv(employees_missing, "employees_missing.csv")
```

The file contains:
- Empty cells
- The string `"N/A"`
- The string `"NULL"`
- The sentinel value `-999`

**Default import — readr only recognises `""` and `"NA"`:**

```r
df_default <- read_csv("employees_missing.csv")
# Warning: One or more parsing issues.

problems(df_default)
#   row   col  expected    actual
#     2     4  a double    N/A
#     6     4  a double    NULL
```

- `N/A` and `NULL` cause **parsing failures** in the numeric salary column
- readr **warns you** — base R would have silently converted the whole column to character

**The fix — declare all missing-value representations:**

```r
df_clean <- read_csv("employees_missing.csv",
                     na = c("", "NA", "N/A", "NULL", "-999"))

df_clean$salary
# [1] 72000    NA 95000    NA 71000    NA

class(df_clean$salary)
# [1] "numeric"

mean(df_clean$salary, na.rm = TRUE)
# [1] 79333.33

# Count missing values per column
colSums(is.na(df_clean))
# employee_id  first_name  department      salary
#           0           0           1           3
```

> **Always run `problems(df)` after importing a new file.**  
> Zero rows = clean import. Any rows = values that failed to parse.

### 5.3 — Type Guessing Failures

**The scenario:** A column looks numeric for the first 1,000 rows but contains text values further into the file.

```r
# Create a dataset where salary has some text mixed in
employees_mixed <- data.frame(
  employee_id = 1:6,
  first_name  = c("Alice", "Bob", "Carol", "David", "Eve", "Frank"),
  salary      = c(72000, 65000, "PENDING", 68000, 71000, "TBC"),
  stringsAsFactors = FALSE
)
write_csv(employees_mixed, "employees_mixed.csv")
```

**What happens:**

```r
df <- read_csv("employees_mixed.csv")
# Warning: 2 parsing failures.
# readr guessed salary as numeric, then hit "PENDING" and "TBC"
# It replaced those cells with NA

df$salary
# [1] 72000 65000    NA 68000 71000    NA
```

**The fix — read as character first, then investigate:**

```r
df <- read_csv("employees_mixed.csv",
  col_types = cols(salary = col_character())
)

# Find the non-numeric values
df$salary[!grepl("^[0-9.]+$", df$salary)]
# [1] "PENDING" "TBC"

# Replace them with NA, then convert to numeric
df$salary[df$salary %in% c("PENDING", "TBC")] <- NA
df$salary <- as.numeric(df$salary)

df$salary
# [1] 72000 65000    NA 68000 71000    NA
```

### 5.4 — Date Format Issues

**The scenario:** Dates in the file use a non-standard format like `15/03/2024` (day/month/year).

```r
# Create a dataset with UK-style dates
employees_dates <- employees
employees_dates$hire_date <- c("15/03/2019", "01/07/2021", "20/11/2018",
                                "10/05/2020", "30/01/2022", "14/08/2017")
write_csv(employees_dates, "employees_dates.csv")
```

```r
# Default import — readr reads non-ISO dates as character
df <- read_csv("employees_dates.csv")
class(df$hire_date)   # [1] "character"

# Fix: tell readr the exact format
df <- read_csv("employees_dates.csv",
  col_types = cols(
    hire_date = col_date(format = "%d/%m/%Y")
  )
)

class(df$hire_date)   # [1] "Date"
df$hire_date[1]       # [1] "2019-03-15"
```

**Common date format codes:**

| Code | Meaning | Example |
|---|---|---|
| `%Y` | 4-digit year | 2024 |
| `%y` | 2-digit year | 24 |
| `%m` | numeric month | 03 |
| `%d` | day of month | 15 |
| `%B` | full month name | March |
| `%b` | abbreviated month name | Mar |

### 5.5 — File Encoding Issues

**The scenario:** Files from European or older Windows systems may use non-UTF-8 encoding. Special characters (accented letters, currency symbols) appear corrupted.

```r
# Detect the encoding of a file before reading
guess_encoding("employees_fr.csv")
#   encoding   confidence
#   ISO-8859-1       0.85

# Specify encoding explicitly via locale
df <- read_csv("employees_fr.csv",
               locale = locale(encoding = "ISO-8859-1"))
```

> **Tip:** UTF-8 is the modern standard. If you can control the source file,  
> always save it as UTF-8 to avoid encoding problems entirely.

## 6. readr's Companion Functions

All readr functions share the **same arguments** as `read_csv()` — only the default delimiter changes.

| Function | Default delimiter | Use it for |
|---|---|---|
| `read_csv()` | `,` | Standard CSV — the most common format |
| `read_csv2()` | `;` | European CSV where `,` is the decimal mark |
| `read_tsv()` | `\t` (tab) | Tab-separated exports from Excel or databases |
| `read_delim()` | user-defined | Any other delimiter — maximum flexibility |
| `read_fwf()` | fixed positions | Fixed-width files from legacy systems |
| `read_lines()` | — | Read file line-by-line as a character vector |
| `read_file()` | — | Read entire file as a single string |

### Examples using the same employees data

```r
# Save as tab-separated
write_tsv(employees, "employees.tsv")

# Save as pipe-separated
write_delim(employees, "employees.txt", delim = "|")
```

```r
# Read tab-separated
df_tsv  <- read_tsv("employees.tsv")

# Read semicolon-separated (European CSV)
df_semi <- read_csv2("employees_eu.csv")

# Read pipe-separated
df_pipe <- read_delim("employees.txt", delim = "|")

# All functions take the same arguments
df_pipe <- read_delim("employees.txt",
                      delim          = "|",
                      na             = c("", "NA", "NULL"),
                      skip           = 2,
                      n_max          = 100,
                      show_col_types = FALSE)
```

## 7. read.csv() vs read_csv() — Quick Comparison

| Feature | `read.csv()` base R | `read_csv()` readr |
|---|---|---|
| Output type | `data.frame` | `tibble` |
| Column names with spaces | Silently renamed with dots | Preserved exactly, use backticks |
| Default missing values | `"NA"` only | `""` and `"NA"` |
| Column type feedback | Silent | Prints column spec to console |
| Parsing failures | Silently coerces type | Warns + `problems()` report |
| Speed on large files | Slower (pure R) | Faster (C++ backend) |
| Strings to factors | FALSE since R 4.0 | Never |
| URL support | Basic | Yes |
| Select columns on import | Not available | `col_select` argument |
| Retrieve column spec | Not available | `spec()` function |
| Package required | None | `readr` |

## Quick Reference — Cheat Sheet

```r
library(readr)

# Basic read
df <- read_csv("file.csv")

# Suppress column spec message
df <- read_csv("file.csv", show_col_types = FALSE)

# Handle multiple missing value codes
df <- read_csv("file.csv", na = c("", "NA", "N/A", "NULL", "-999"))

# Explicit column types
df <- read_csv("file.csv",
  col_types = cols(
    id        = col_integer(),
    name      = col_character(),
    category  = col_factor(),
    amount    = col_double(),
    date      = col_date(format = "%Y-%m-%d")
  )
)

# Read only specific columns
df <- read_csv("file.csv", col_select = c(id, name, amount))

# Skip header rows and limit rows read
df <- read_csv("file.csv", skip = 3, n_max = 500)

# Non-UTF-8 encoding
df <- read_csv("file.csv", locale = locale(encoding = "ISO-8859-1"))

# Audit every import
problems(df)          # see parsing failures
spec(df)              # see column specification
colSums(is.na(df))    # count missing values per column
```
