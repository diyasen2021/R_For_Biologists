## 1. What is R?

**R** is a free, open-source programming language designed specifically for statistics, data analysis, and data visualisation.

### Where did R come from?

- Created in the early 1990s by **Ross Ihaka** and **Robert Gentleman** at the University of Auckland
- Based on an earlier language called **S**, developed at Bell Labs
- Maintained today by the **R Core Team** and a global community of contributors
- Released under a free licence — it costs nothing and always will

### What is R used for?

- **Data cleaning and transformation** — reshaping, filtering, and preparing raw data
- **Statistical analysis** — everything from a simple mean to complex regression models
- **Data visualisation** — charts, plots, and interactive dashboards
- **Reporting** — combining code, results, and narrative in a single document (R Markdown)
- **Machine learning** — building and evaluating predictive models

### Who uses R?

- Statisticians and data scientists
- Academic researchers across almost every field
- Pharmaceutical and clinical trial teams
- Finance, economics, and government analysts
- Journalists doing data-driven reporting

### R vs Python — a quick comparison

| | R | Python |
|---|---|---|
| **Primary strength** | Statistics and data visualisation | General-purpose programming |
| **Learning curve** | Gentle for data tasks | Steeper for beginners |
| **Visualisation** | ggplot2 is best-in-class | Matplotlib / seaborn |
| **Statistics** | Built-in statistical functions | Requires additional libraries |
| **Community** | Strong in academia and research | Strong in industry and ML |
| **Cost** | Free | Free |

> **Bottom line:** R and Python are both excellent tools.  
> For statistics, analysis, and research, R is often the more natural choice.  
> Many data professionals use both.

## 2. What is RStudio?

**RStudio** is an **Integrated Development Environment (IDE)** for R.

### The difference between R and RStudio

Think of it this way:
- **R** is the engine — the language that actually runs your code
- **RStudio** is the dashboard — a friendly interface for writing, running, and organising your work

You can use R without RStudio (in a plain terminal), but it is much harder.  
RStudio makes the experience dramatically better, and it is what virtually everyone uses.

### What does RStudio give you?

- A **code editor** with syntax highlighting, auto-complete, and error hints
- A **console** to run code interactively
- A **viewer** for plots, data, and documentation — all in one window
- **Project management** — keep all files for an analysis together
- **Version control integration** — connect directly to Git and GitHub
- Tools for writing **R Markdown** reports that mix code and text
- A **package manager** with a point-and-click interface

### RStudio Desktop vs Posit Cloud

| | RStudio Desktop | Posit Cloud (browser) |
|---|---|---|
| **Where it runs** | Your own computer | In a web browser |
| **Installation** | Required | None needed |
| **Internet needed** | Only for packages | Always |
| **Cost** | Free | Free tier available |
| **Best for** | Regular use, large files | Quick start, shared teaching environments |

> **For this course we will use RStudio Desktop.**  
> Posit Cloud (formerly RStudio Cloud) is a good backup if installation causes problems.  
> Visit: [posit.cloud](https://posit.cloud)

## 3. Machine Requirements

### Minimum requirements

| Component | Minimum | Recommended |
|---|---|---|
| **Operating System** | Windows 10, macOS 11, Ubuntu 20.04 | Windows 11, macOS 13+, Ubuntu 22.04 |
| **Processor** | 64-bit, any modern CPU | Multi-core (2+ cores) |
| **RAM** | 4 GB | 8 GB or more |
| **Storage** | 2 GB free space | 5+ GB free space |
| **Screen** | 1024 × 768 | 1920 × 1080 or higher |
| **Internet** | Required for installation and packages | Broadband recommended |

### Notes on RAM

- R loads data **entirely into memory** — RAM matters more than CPU for most tasks
- A dataset with 1 million rows and 20 columns uses roughly **150 MB of RAM**
- For large datasets (millions of rows), 16 GB RAM is strongly recommended
- If RAM is limited, there are packages (`data.table`, `arrow`) that handle larger-than-memory data

### 32-bit vs 64-bit

- R requires a **64-bit operating system** (R 4.0 and later dropped 32-bit support)
- All computers made after 2010 are 64-bit
- To check on Windows: **Start → Settings → System → About → System type**

### macOS and Linux

- This guide focuses on **Windows** installation
- macOS: download the `.pkg` installer from [cran.r-project.org](https://cran.r-project.org) — process is nearly identical
- Linux (Ubuntu/Debian): R can be installed via `apt` — instructions at [cran.r-project.org/bin/linux/ubuntu](https://cran.r-project.org/bin/linux/ubuntu/)

## 4. Installation Guide — Windows

Install in this order: **R first, then RStudio.**  
RStudio needs R to already be present on the machine.

---

### Step 1 — Download R

1. Go to **[cran.r-project.org](https://cran.r-project.org)**
2. Click **"Download R for Windows"**
3. Click **"base"**
4. Click **"Download R-x.x.x for Windows"** (the current version)
5. Run the downloaded `.exe` file

### Step 1 — Install R (installer options)

- Accept the licence agreement
- Leave the **installation path** as the default (`C:\Program Files\R\R-x.x.x`)
- Leave **components** as default (Core Files, 64-bit Files, Message Translations)
- Leave **startup options** as default (No — accept defaults)
- Leave **Start Menu folder** as default
- Click Install, then Finish

> Do **not** change the installation path.  
> Spaces in folder names (e.g. `C:\My Files\R`) can cause obscure errors later.

---

### Step 2 — Download RStudio

1. Go to **[posit.co/download/rstudio-desktop](https://posit.co/download/rstudio-desktop/)**
2. Click **"Download RStudio Desktop"** (the free version)
3. The site should auto-detect Windows — click the **Windows download button**
4. Run the downloaded `.exe` file

### Step 2 — Install RStudio (installer options)

- Accept the licence agreement
- Leave the **installation path** as the default
- Click Install, then Finish
- RStudio will appear in your Start Menu and optionally on your Desktop

---

### Step 3 — Verify the installation

1. Open **RStudio** from the Start Menu
2. In the **Console** panel (bottom left), you should see something like:

```
R version 4.4.1 (2024-06-14) -- "Race for Your Life"
Copyright (C) 2024 The R Foundation for Statistical Computing
Platform: x86_64-w64-mingw32/x64

Type 'demo()' for some demos, 'help()' for on-line help, or
'help.start()' for an HTML browser interface to help.
Type 'q()' to quit R.
>
```

3. Click in the Console and type:

```r
2 + 2
```

4. Press **Enter** — you should see `[1] 4`

If you see `[1] 4`, R and RStudio are installed correctly.

---

### Keeping R and RStudio up to date

- R releases new versions roughly **twice a year**
- RStudio releases updates more frequently
- To check your current R version: type `R.version.string` in the Console
- To update RStudio: **Help → Check for Updates** in the menu bar
- To update R on Windows, use the `installr` package:

```r
install.packages("installr")
installr::updateR()
```

## 5. A Tour of RStudio

When you first open RStudio you will see **four panels**.  
Each panel has a specific purpose. You can resize them by dragging the borders.

```
┌─────────────────────────┬─────────────────────────┐
│                         │                         │
│    Source Editor        │    Environment /        │
│    (top left)           │    History              │
│                         │    (top right)          │
│                         │                         │
├─────────────────────────┼─────────────────────────┤
│                         │                         │
│    Console /            │    Files / Plots /      │
│    Terminal             │    Packages / Help      │
│    (bottom left)        │    (bottom right)       │
│                         │                         │
└─────────────────────────┴─────────────────────────┘
```

---

### Panel 1 — The Source Editor (top left)

This is where you **write and save your R scripts**.

- A script is simply a text file of R code (saved as `.R`)
- You can have multiple scripts open as tabs
- Code in the editor does **not** run automatically — you send it to the Console
- **Run a single line:** put your cursor on the line and press `Ctrl + Enter`
- **Run a selection:** highlight multiple lines and press `Ctrl + Enter`
- **Run the whole script:** press `Ctrl + Shift + Enter`
- Lines starting with `#` are **comments** — R ignores them, they are notes for humans

```r
# This is a comment — R ignores this line
x <- 10        # assign the value 10 to x
y <- 20        # assign the value 20 to y
x + y          # add them — press Ctrl+Enter to run this line
```

> **Always write code in the Source Editor, not directly in the Console.**  
> Code in the Console disappears when you close RStudio. Scripts are saved files.

---

### Panel 2 — The Console (bottom left)

This is where R **actually runs your code** and shows output.

- When you press `Ctrl + Enter` in the editor, code is sent here and executed
- You can also type directly in the Console for quick, one-off commands
- The `>` symbol is the **prompt** — R is waiting for input
- The `+` symbol means R is waiting for you to finish an incomplete expression
- Output appears directly below the command that produced it
- **Error messages** appear in red — read them carefully, they usually tell you exactly what went wrong

```
> x <- 10
> y <- 20
> x + y
[1] 30
> mean(c(1, 2, 3, NA))
[1] NA
> mean(c(1, 2, 3, NA), na.rm = TRUE)
[1] 2
```

**Console keyboard shortcuts:**

| Shortcut | Action |
|---|---|
| `↑` arrow key | Recall previous command |
| `Ctrl + L` | Clear the console display |
| `Escape` | Cancel a running command |
| `Tab` | Auto-complete a function or object name |

---

### Panel 3 — Environment / History (top right)

#### Environment tab

- Shows every **object currently in memory** — variables, data frames, functions you have created
- Tells you the type, dimensions, and a preview of each object
- Click the broom icon 🧹 to clear all objects from memory
- Click on a data frame name to open it in a viewer (like a spreadsheet)

```
Values
  x       10
  y       20

Data
  flights  336776 obs. of 19 variables
```

- When you close RStudio, you will be asked whether to **save the workspace** (all objects in the Environment)
- **Recommendation: always click No** — your script should recreate everything from scratch when re-run
- Saving the workspace creates hidden `.RData` files and leads to hard-to-reproduce results

#### History tab

- Shows every command you have run in the Console, in order
- Useful for recovering something you typed but did not save
- You can search, select, and send historical commands back to the editor

---

### Panel 4 — Files / Plots / Packages / Help (bottom right)

This panel has multiple tabs — each does something different.

#### Files tab

- A file browser showing the contents of your **working directory**
- Use it to navigate folders, open files, and rename or delete files
- Easier than remembering file paths when you are starting out
- Click the **gear icon** to set the working directory to the current folder

#### Plots tab

- Every plot you create appears here automatically
- Use the **left and right arrows** to navigate through previous plots
- Click **Zoom** to open the plot in a larger window
- Click **Export** to save the plot as PNG, PDF, or copy it to the clipboard

#### Packages tab

- Shows all installed packages and whether they are currently loaded
- Tick a package to load it (equivalent to `library(package_name)`)
- Click **Install** to search for and install new packages from CRAN
- Click **Update** to update packages to their latest versions

> **Important:** loading a package by ticking it in the GUI works once but is not reproducible.  
> Always write `library(package_name)` at the top of your script instead.

#### Help tab

- Displays documentation for any R function
- Access it by typing `?function_name` in the Console:

```r
?mean          # opens help page for mean()
?read_csv      # opens help page for read_csv()
help("filter") # alternative syntax
```

- Every help page has the same structure: **Description, Usage, Arguments, Value, Examples**
- Scroll to the **Examples** section at the bottom — these are usually the fastest way to learn a function

---

### Useful RStudio settings to change once

Go to **Tools → Global Options** and consider these settings:

| Setting | Where to find it | Recommended value |
|---|---|---|
| Save workspace to `.RData` on exit | General → Basic | **Never** |
| Restore `.RData` into workspace at startup | General → Basic | **Unticked** |
| Soft-wrap R source files | Code → Editing | **Ticked** |
| Rainbow parentheses | Code → Display | **Ticked** — helps spot mismatched brackets |
| Show margin column at 80 characters | Code → Display | **Ticked** — keeps code readable |
| Default text encoding | Code → Saving | **UTF-8** |

## 6. Your First R Commands

Open a new script with **Ctrl + Shift + N** and try these:

### Arithmetic

```r
# Basic arithmetic
5 + 3       # [1] 8
10 - 4      # [1] 6
6 * 7       # [1] 42
15 / 4      # [1] 3.75
15 %/% 4    # [1] 3    — integer division (whole number part only)
15 %% 4     # [1] 3    — modulo (remainder)
2 ^ 8       # [1] 256  — exponentiation
sqrt(144)   # [1] 12
```

### Assigning variables

```r
# <- is the assignment operator (keyboard shortcut: Alt + -)
x <- 42
name <- "Alice"
is_enrolled <- TRUE

# Print by just typing the name
x             # [1] 42
name          # [1] "Alice"

# Or use print() explicitly
print(is_enrolled)   # [1] TRUE
```

### Vectors — R's basic building block

```r
# c() combines values into a vector
ages <- c(23, 31, 28, 45, 19)
names_vec <- c("Alice", "Bob", "Carol", "David", "Eve")

# Operations apply to every element
ages + 1           # [1] 24 32 29 46 20
ages * 2           # [1] 46 62 56 90 38
ages > 25          # [1] FALSE  TRUE  TRUE  TRUE FALSE

# Useful vector functions
length(ages)       # [1] 5
sum(ages)          # [1] 146
mean(ages)         # [1] 29.2
min(ages)          # [1] 19
max(ages)          # [1] 45
sort(ages)         # [1] 19 23 28 31 45
```

### Your first data frame

```r
# Create a small data frame by hand
students <- data.frame(
  name    = c("Alice", "Bob", "Carol", "David"),
  age     = c(23, 31, 28, 45),
  score   = c(88, 74, 91, 65),
  passed  = c(TRUE, TRUE, TRUE, FALSE)
)

# Print it
students

# Access a column with $
students$score       # [1] 88 74 91 65

# Basic summary
summary(students)
nrow(students)       # [1] 4
ncol(students)       # [1] 4
```

## 7. Understanding Packages

### What is a package?

- R comes with a set of built-in functions — this is called **base R**
- A **package** is a collection of additional functions someone else has written and shared
- There are over **20,000 packages** on CRAN (the official R package repository)
- Packages are free to install and use

### The two-step process

```r
# Step 1 — Install the package (done ONCE, ever)
# Downloads files from the internet to your computer
install.packages("readr")

# Step 2 — Load the package (done ONCE per session, at the top of your script)
# Makes the package's functions available in your current session
library(readr)
```

> Think of it this way:  
> `install.packages()` = buying a book and putting it on your shelf  
> `library()` = taking the book off the shelf and opening it

### The tidyverse — a bundle of essential packages

The **tidyverse** is a curated collection of packages that work together and share a consistent design.  
Installing the tidyverse gives you everything you need for most data analysis work.

```r
# Install once — takes a few minutes, downloads several packages
install.packages("tidyverse")

# Load the core tidyverse packages
library(tidyverse)

# The main packages loaded by library(tidyverse):
# readr    — reading data files
# dplyr    — data transformation
# ggplot2  — data visualisation
# tidyr    — reshaping data
# tibble   — modern data frames
# stringr  — working with text
# forcats  — working with factors
# purrr    — functional programming
```

### Common beginner package mistakes

| Mistake | What happens | Fix |
|---|---|---|
| `library()` without `install.packages()` first | Error: there is no package called '...' | Run `install.packages("name")` first |
| `install.packages()` inside the script | Reinstalls every time you run the script | Put it in the Console once, never in your script |
| Forgetting `library()` at the top of a script | Error: could not find function "..." | Add `library(package)` to the top of the script |
| Wrong capitalisation | `Library(dplyr)` fails | R is case-sensitive — always lowercase `library` |

## 8. Good Habits from Day One

These habits will save you hours of frustration. Start them now.

---

### Use RStudio Projects

An **RStudio Project** keeps all files for one piece of work — scripts, data, outputs — in one folder, and sets the working directory automatically.

```
To create a project:
File → New Project → New Directory → New Project
Give it a name, choose a folder, click Create Project
```

- Your project opens in a clean session every time
- File paths become simple — `"data/myfile.csv"` instead of `"C:/Users/Alice/Documents/uni/semester2/stats/data/myfile.csv"`
- Easy to share — zip the folder and send it, and it works on someone else's machine

---

### Structure your script files

```r
# ============================================================
# Script:   01_data_cleaning.R
# Purpose:  Load and clean the survey data
# Author:   Your Name
# Date:     2024-03-01
# ============================================================

# ── 1. Load packages ─────────────────────────────────────────
library(readr)
library(dplyr)

# ── 2. Read data ──────────────────────────────────────────────
df <- read_csv("data/survey_raw.csv")

# ── 3. Clean data ─────────────────────────────────────────────
# ... your code here
```

- A clear header tells anyone (including future you) what the script does
- Group code into labelled sections
- Use comments to explain *why*, not just *what*

---

### Use a consistent folder structure

```
my_project/
├── data/
│   ├── raw/          ← original files, never modified
│   └── processed/    ← cleaned versions your scripts produce
├── scripts/
│   ├── 01_clean.R
│   ├── 02_analyse.R
│   └── 03_visualise.R
├── outputs/
│   ├── figures/
│   └── tables/
└── my_project.Rproj
```

### Keyboard shortcuts worth learning immediately

| Shortcut | Action |
|---|---|
| `Ctrl + Enter` | Run current line or selection |
| `Ctrl + Shift + Enter` | Run entire script |
| `Ctrl + Shift + N` | New script |
| `Ctrl + S` | Save script |
| `Alt + -` | Insert `<-` (assignment operator) |
| `Ctrl + Shift + C` | Comment / uncomment selected lines |
| `Ctrl + Shift + R` | Insert a section divider comment |
| `Tab` | Auto-complete |

---

### Summary — the workflow for every session

```
1. Open RStudio → open your Project
2. Open or create a script (Ctrl + Shift + N)
3. Write library() calls at the top
4. Write your code in the script — not the Console
5. Run lines with Ctrl + Enter as you go
6. Save your script frequently (Ctrl + S)
7. When done: close RStudio → click "Don't Save" if asked about the workspace
```
