## 1. What is dplyr?

**dplyr** is the tidyverse package for transforming and summarising data.  
It gives you a small set of functions — called **verbs** — that cover the vast majority of data manipulation tasks.

### Why dplyr over base R?

- Code reads like plain English — `filter()`, `select()`, `arrange()` say exactly what they do
- Works seamlessly with the **pipe operator** — chain steps together without nested brackets
- Consistent behaviour — every verb takes a data frame in and returns a data frame out
- Fast — optimised for performance on large datasets

### The four verb groups

dplyr verbs are organised by **what they operate on**:

| Group | Verbs | What they do |
|---|---|---|
| **Rows** | `filter()`, `arrange()`, `slice()` | Keep, remove, or reorder rows |
| **Columns** | `select()`, `rename()`, `mutate()`, `relocate()` | Keep, rename, create, or reorder columns |
| **Groups** | `group_by()`, `ungroup()`, `summarise()` | Aggregate and summarise by category |
| **Tables** | `left_join()`, `inner_join()`, `bind_rows()` | Combine multiple data frames |

> We will cover Rows, Columns, and Groups in this tutorial.  
> Table verbs (joins) will be covered in a later session.

## 3. The Pipe Operator

### The problem without pipes

Suppose you want to:
1. Filter to only **Gentoo** penguins
2. Then keep only the `species`, `body_mass_g`, and `flipper_length_mm` columns
3. Then sort by body mass descending

Without a pipe, you have two options — both are awkward:

```r
# Option A — nested functions (read inside-out, hard to follow)
arrange(
  select(
    filter(penguins, species == "Gentoo"),
    species, body_mass_g, flipper_length_mm
  ),
  desc(body_mass_g)
)

# Option B — intermediate objects (clutters your environment)
step1 <- filter(penguins, species == "Gentoo")
step2 <- select(step1, species, body_mass_g, flipper_length_mm)
step3 <- arrange(step2, desc(body_mass_g))
```

### The pipe solves this

The pipe operator **`|>`** takes the result on the left and passes it as the first argument to the function on the right.

```
data |> function()   is the same as   function(data)
```

```r
# The same three steps — now readable top to bottom
penguins |>
  filter(species == "Gentoo") |>
  select(species, body_mass_g, flipper_length_mm) |>
  arrange(desc(body_mass_g))
```

- Read it as: *"take penguins, then filter to Gentoo, then select those columns, then sort"*
- Each line does one thing
- Easy to add, remove, or reorder steps
- The result is only stored if you assign it: `result <- penguins |> filter(...)`

### The two pipe operators you will see

```r
# |>  — the native R pipe (R 4.1+), no packages needed
penguins |> filter(species == "Gentoo")

# %>%  — the magrittr pipe from the tidyverse (older code uses this)
penguins %>% filter(species == "Gentoo")
```

> For new code, use `|>`.  
> You will see `%>%` in older tutorials and Stack Overflow answers — they behave the same for our purposes.

## 4. Row Verbs — filter() and arrange()

Row verbs work on **rows** — they never add, remove, or rename columns.

---

### `filter()` — keep rows that match a condition

```r
# Single condition — one species only
penguins |> filter(species == "Chinstrap")
```

```
# A tibble: 68 × 8
   species   island bill_length_mm bill_depth_mm flipper_length_mm body_mass_g sex     year
   <fct>     <fct>           <dbl>         <dbl>             <int>       <int> <fct>  <int>
 1 Chinstrap Dream            46.5          17.9               192        3500 female  2007
 2 Chinstrap Dream            50.0          19.5               196        3900 male    2007
 3 Chinstrap Dream            51.3          19.2               193        3650 male    2007
...
```

```r
# Multiple conditions — AND (both must be true)
# Female Gentoo penguins only
penguins |> filter(species == "Gentoo", sex == "female")

# Multiple conditions — OR (either can be true)
# Adélie OR Chinstrap penguins
penguins |> filter(species == "Adelie" | species == "Chinstrap")

# Equivalent shorthand for OR on the same column
penguins |> filter(species %in% c("Adelie", "Chinstrap"))

# NOT — exclude one species
penguins |> filter(species != "Gentoo")

# Numeric comparisons — large penguins only
penguins |> filter(body_mass_g >= 5000)       # 5 kg or heavier
penguins |> filter(flipper_length_mm > 210)   # long flippers
penguins |> filter(bill_length_mm < 35)       # short bills

# Combining conditions — heavy penguins from Biscoe island
penguins |> filter(island == "Biscoe", body_mass_g > 5000)

# Penguins with a long bill relative to its depth
penguins |> filter(bill_length_mm > 2 * bill_depth_mm)

# Filter on missing values
penguins |> filter(is.na(sex))         # penguins with no recorded sex (11 rows)
penguins |> filter(!is.na(sex))        # only penguins with sex recorded
```

### Comparison operators — quick reference

| Operator | Meaning | Example |
|---|---|---|
| `==` | equal to | `species == "Gentoo"` |
| `!=` | not equal to | `island != "Dream"` |
| `>` | greater than | `body_mass_g > 5000` |
| `>=` | greater than or equal | `flipper_length_mm >= 200` |
| `<` | less than | `bill_length_mm < 35` |
| `<=` | less than or equal | `bill_depth_mm <= 15` |
| `%in%` | is one of these values | `species %in% c("Adelie", "Gentoo")` |
| `!` | NOT | `!is.na(sex)` |
| `&` or `,` | AND | `species == "Gentoo", sex == "male"` |
| `\|` | OR | `island == "Biscoe" \| island == "Dream"` |

## 5. Column Verbs — select(), rename(), mutate()

Column verbs work on **columns** — they never add or remove rows.

---

### `select()` — keep or drop columns

```r
# Keep specific columns by name
penguins |> select(species, island, body_mass_g, sex)

# Keep a range of adjacent columns
penguins |> select(bill_length_mm:body_mass_g)

# Drop columns with -
penguins |> select(-year)
penguins |> select(-c(year, island))

# Reorder — put species and sex first, keep everything else
penguins |> select(species, sex, everything())
```

### Handy helper functions inside `select()`

```r
# Columns whose name starts with a string
penguins |> select(starts_with("bill"))      # bill_length_mm, bill_depth_mm

# Columns whose name ends with a string
penguins |> select(ends_with("mm"))          # bill_length_mm, bill_depth_mm, flipper_length_mm

# Columns whose name contains a string
penguins |> select(contains("length"))       # bill_length_mm, flipper_length_mm

# Select only numeric columns
penguins |> select(where(is.numeric))        # bill_length_mm, bill_depth_mm, flipper_length_mm, body_mass_g, year

# Select only factor columns
penguins |> select(where(is.factor))         # species, island, sex
```

---

### `rename()` — rename columns

```r
# rename(new_name = old_name)
penguins |> rename(
  mass_g         = body_mass_g,
  flipper_mm     = flipper_length_mm,
  bill_length    = bill_length_mm,
  bill_depth     = bill_depth_mm
)
```

> `rename()` only changes names — it never drops columns.  
> `select()` can also rename while selecting: `select(mass_g = body_mass_g, species)`

---

### `mutate()` — add or change columns

`mutate()` adds new columns or overwrites existing ones.  
New columns appear at the right side of the tibble.

```r
# Convert body mass from grams to kilograms
penguins |>
  mutate(body_mass_kg = body_mass_g / 1000) |>
  select(species, sex, body_mass_g, body_mass_kg)
```

```
# A tibble: 344 × 4
   species sex    body_mass_g body_mass_kg
   <fct>   <fct>        <int>        <dbl>
 1 Adelie  male          3750         3.75
 2 Adelie  female        3800         3.80
 3 Adelie  female        3250         3.25
 4 Adelie  NA              NA           NA
...
```

```r
# Add multiple columns at once — and use a new column immediately
penguins |>
  mutate(
    body_mass_kg   = body_mass_g / 1000,
    bill_ratio     = bill_length_mm / bill_depth_mm,    # bill shape index
    flipper_cm     = flipper_length_mm / 10
  ) |>
  select(species, body_mass_kg, bill_ratio, flipper_cm)

# Conditional column — classify by size
penguins |>
  mutate(
    size_class = ifelse(body_mass_g >= 4500, "Large", "Small")
  ) |>
  select(species, body_mass_g, size_class)

# Multiple categories with case_when() — classify bill shape ecologically
# Long thin bills = fish feeders, short deep bills = krill/crustacean feeders
penguins |>
  mutate(
    bill_shape = case_when(
      bill_ratio >= 3.0            ~ "Long and thin",
      bill_ratio >= 2.5            ~ "Intermediate",
      bill_ratio <  2.5            ~ "Short and deep"
    ),
    .after = bill_depth_mm   # place the new column right after bill_depth_mm
  )
```

```
# A tibble: 344 × 9
   species   island    bill_length_mm bill_depth_mm bill_shape     flipper_length_mm ...
   <fct>     <fct>              <dbl>         <dbl> <chr>                      <int>
 1 Adelie    Torgersen           39.1          18.7 Intermediate                 181
 2 Adelie    Torgersen           39.5          17.4 Intermediate                 186
 3 Chinstrap Dream               46.5          17.9 Intermediate                 192
 4 Gentoo    Biscoe              50.0          14.2 Long and thin                230
...
```

> **Biological note:** Bill shape relates directly to diet.  
> Gentoo penguins tend to have longer, thinner bills suited for catching fish.  
> Adélie penguins have shorter, deeper bills suited for crustaceans and krill.

## 6. Group Verbs — group_by() and summarise()

This is where dplyr becomes genuinely powerful.  
Instead of operating on the whole dataset, you can split it into groups and run calculations **per group**.

---

### `summarise()` — collapse rows into summary statistics

Without grouping, `summarise()` collapses the whole dataset into one row:

```r
penguins |>
  summarise(
    total_penguins     = n(),
    mean_body_mass_g   = mean(body_mass_g, na.rm = TRUE),
    mean_flipper_mm    = mean(flipper_length_mm, na.rm = TRUE),
    mean_bill_length   = mean(bill_length_mm, na.rm = TRUE)
  )
```

```
# A tibble: 1 × 4
  total_penguins mean_body_mass_g mean_flipper_mm mean_bill_length
           <int>            <dbl>           <dbl>            <dbl>
1            344            4202.            201.             43.9
```

### Common summary functions

| Function | What it calculates |
|---|---|
| `n()` | count of rows |
| `n_distinct(x)` | count of unique values |
| `sum(x)` | total |
| `mean(x)` | average |
| `median(x)` | median |
| `min(x)` | minimum |
| `max(x)` | maximum |
| `sd(x)` | standard deviation |
| `first(x)` | first value in the group |
| `last(x)` | last value in the group |

> Always add `na.rm = TRUE` inside numeric functions — `penguins` has NAs in measurement columns.  
> Example: `mean(body_mass_g, na.rm = TRUE)`

---

### `group_by()` — split into groups before summarising

`group_by()` does not change the data — it attaches grouping information.  
The magic happens when you chain it with `summarise()`.

```r
# Mean body mass and flipper length per species
penguins |>
  group_by(species) |>
  summarise(
    n               = n(),
    mean_mass_g     = round(mean(body_mass_g, na.rm = TRUE)),
    mean_flipper_mm = round(mean(flipper_length_mm, na.rm = TRUE), 1),
    mean_bill_mm    = round(mean(bill_length_mm, na.rm = TRUE), 1)
  ) |>
  ungroup()
```

```
# A tibble: 3 × 5
  species       n mean_mass_g mean_flipper_mm mean_bill_mm
  <fct>     <int>       <dbl>           <dbl>        <dbl>
1 Adelie      152        3701             190.         38.8
2 Chinstrap    68        3733             196.         48.8
3 Gentoo      124        5076             217.         47.5
```

> **Biological insight:** Gentoo penguins are substantially heavier and have longer flippers  
> than Adélie or Chinstrap — consistent with their larger body size and deeper-diving lifestyle.

```r
# Group by multiple columns — species AND sex
penguins |>
  group_by(species, sex) |>
  summarise(
    n           = n(),
    mean_mass_g = round(mean(body_mass_g, na.rm = TRUE)),
    sd_mass_g   = round(sd(body_mass_g, na.rm = TRUE))

```

### `group_by()` + `mutate()` — add a group calculation back to the original rows

Sometimes you want the group summary **alongside the original rows**, not collapsed into one row per group.  
Use `mutate()` instead of `summarise()` after `group_by()`.

```r
# Add each species' mean body mass alongside every individual penguin row
# Useful for comparing each individual to its species average
penguins |>
  group_by(species) |>
  mutate(
    species_mean_mass = mean(body_mass_g, na.rm = TRUE),
    diff_from_mean    = body_mass_g - species_mean_mass
  ) |>
  select(species, sex, body_mass_g, species_mean_mass, diff_from_mean) |>
  ungroup()
```

```
# A tibble: 344 × 5
# Groups: species [3]
   species sex    body_mass_g species_mean_mass diff_from_mean
   <fct>   <fct>        <int>             <dbl>          <dbl>
 1 Adelie  male          3750             3701.           49.3   ← 49g above Adelie average
 2 Adelie  female        3800             3701.           99.3
 3 Adelie  female        3250             3701.         -451.    ← 451g below Adelie average
 4 Adelie  NA              NA             3701.           NA
...
```

> This pattern is very useful in biology — it lets you ask  
> *"is this individual large or small relative to its own species?"*  
> rather than comparing it to the overall dataset average.

---

### `ungroup()` — always clean up after grouping

After `group_by()`, the tibble stays grouped until you explicitly remove the grouping.  
Forgetting to `ungroup()` can cause unexpected results in later steps.

```r
# Safe pattern — always ungroup when done
penguins |>
  group_by(species) |>
  summarise(mean_mass = mean(body_mass_g, na.rm = TRUE)) |>
  ungroup()

# Check whether a tibble is currently grouped
penguins_grouped <- penguins |> group_by(species)
is.grouped_df(penguins_grouped)     # TRUE

penguins_ungrouped <- penguins_grouped |> ungroup()
is.grouped_df(penguins_ungrouped)   # FALSE
```

> **Rule of thumb:** if you assign a grouped result to a variable, always `ungroup()` at the end.

## 7. Putting It All Together

Real biological analysis chains multiple verbs together.  
Here are three realistic questions answered using `palmerpenguins`.

---

### Example 1 — Which species shows the strongest sexual dimorphism in body mass?

```r
penguins |>
  filter(!is.na(sex), !is.na(body_mass_g)) |>
  group_by(species, sex) |>
  summarise(
    n           = n(),
    mean_mass_g = round(mean(body_mass_g))
  ) |>
  ungroup() |>
  # Calculate the male-female difference per species
  group_by(species) |>
  mutate(
    dimorphism_g   = mean_mass_g - mean_mass_g[sex == "female"],
    dimorphism_pct = round((mean_mass_g / mean_mass_g[sex == "female"] - 1) * 100, 1)
  ) |>
  filter(sex == "male") |>
  select(species, mean_mass_g, dimorphism_g, dimorphism_pct) |>
  arrange(desc(dimorphism_pct)) |>
  ungroup()
```

```
# A tibble: 3 × 4
  species   mean_mass_g dimorphism_g dimorphism_pct
  <fct>           <dbl>        <dbl>          <dbl>
1 Gentoo           5485          805           17.2   ← strongest dimorphism
2 Chinstrap        3939          412           11.7
3 Adelie           4043          674           20.0   ← also strong
```

### Example 2 — How do bill dimensions differ across species and islands?

```r
penguins |>
  filter(!is.na(bill_length_mm), !is.na(bill_depth_mm)) |>
  mutate(bill_ratio = round(bill_length_mm / bill_depth_mm, 2)) |>
  group_by(species, island) |>
  summarise(
    n              = n(),
    mean_bill_len  = round(mean(bill_length_mm), 1),
    mean_bill_dep  = round(mean(bill_depth_mm), 1),
    mean_bill_ratio = round(mean(bill_ratio), 2)
  ) |>
  arrange(species, island) |>
  ungroup()
```

```
4 Chinstrap Dream        68          48.8          18.4            2.65
5 Gentoo    Biscoe      119          47.6          14.9            3.20   ← distinctly thin bills
```

> **Biological insight:** Gentoo penguins have a markedly higher bill ratio (longer relative to depth)  
> than Adélie or Chinstrap — a morphological difference that likely reflects dietary specialisation.

## 8. dplyr Cheat Sheet

```r
library(dplyr)
library(palmerpenguins)

# ── Row verbs ─────────────────────────────────────────────────────────
penguins |> filter(species == "Gentoo")                        # keep matching rows
penguins |> filter(species %in% c("Adelie", "Chinstrap"))      # keep rows in a list
penguins |> filter(!is.na(sex))                                # remove NA rows
penguins |> filter(species == "Gentoo", sex == "male")         # AND condition
penguins |> filter(island == "Biscoe" | island == "Dream")     # OR condition
penguins |> arrange(body_mass_g)                               # sort ascending
penguins |> arrange(desc(body_mass_g))                         # sort descending
penguins |> arrange(species, desc(body_mass_g))                # sort by multiple columns
penguins |> slice_max(body_mass_g, n = 5)                      # top 5 heaviest
penguins |> slice_min(body_mass_g, n = 5)                      # bottom 5 lightest

# ── Column verbs ──────────────────────────────────────────────────────
penguins |> select(species, island, body_mass_g)               # keep columns
penguins |> select(-year)                                      # drop a column
penguins |> select(starts_with("bill"))                        # columns starting with "bill"
penguins |> select(ends_with("mm"))                            # columns ending with "mm"
penguins |> select(where(is.numeric))                          # numeric columns only
penguins |> rename(mass_g = body_mass_g)                       # rename a column
penguins |> mutate(mass_kg = body_mass_g / 1000)               # add a column
penguins |> mutate(bill_ratio = bill_length_mm / bill_depth_mm) # calculated column
penguins |> mutate(                                            # conditional column
  size_class = case_when(
    body_mass_g >= 5000 ~ "Large",
    body_mass_g >= 3500 ~ "Medium",
    body_mass_g <  3500 ~ "Small"
  )
)

# ── Group verbs ───────────────────────────────────────────────────────
penguins |>
  group_by(species) |>
  summarise(
    n            = n(),
    mean_mass    = mean(body_mass_g, na.rm = TRUE),
    mean_flipper = mean(flipper_length_mm, na.rm = TRUE),
    sd_mass      = sd(body_mass_g, na.rm = TRUE)
  ) |>
  ungroup()

# group_by + mutate (keeps all rows, adds group calculation alongside them)
penguins |>
  group_by(species) |>
  mutate(species_mean_mass = mean(body_mass_g, na.rm = TRUE)) |>
  ungroup()

# ── Full pipeline reminder ────────────────────────────────────────────
result <- penguins |>
  filter(!is.na(sex), !is.na(body_mass_g)) |>   # step 1 — rows
  select(species, sex, body_mass_g) |>           # step 2 — columns
  mutate(mass_kg = body_mass_g / 1000) |>        # step 3 — new columns
  group_by(species, sex) |>                      # step 4 — grouping
  summarise(mean_mass_kg = mean(mass_kg)) |>     # step 5 — aggregate
  arrange(species, sex) |>                       # step 6 — sort
  ungroup()                                      # always finish with ungroup()
```
