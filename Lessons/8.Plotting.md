## Part 1 — The Grammar of Graphics: Building Layer by Layer

ggplot2 is built on the **Grammar of Graphics** — the idea that every plot can be described by a small set of components stacked on top of each other. You build plots incrementally, one layer at a time, using the `+` operator.

---

### Layer 1: Just the data

We start by calling `ggplot()` with only the `data` argument. This produces a completely blank grey canvas.

```r
ggplot(data = penguins)
```

Nothing appears — we have a blank canvas. ggplot knows *what* data to use, but has no instructions on what to draw or where to put things. We need to add more layers.

---

### Layer 2: Aesthetics — mapping variables to visual properties

**Aesthetics** (`aes`) are **mappings** from variables in your data to **visual properties** of the plot. They answer the questions:

- Which column goes on the x-axis?
- Which column controls the colour of the points?
- Which column controls the size of the marks?

In other words, aesthetics are the bridge between your data and what you see on screen. They always go **inside** `aes()`.

The most common aesthetics are:

| Aesthetic | Controls |
|---|---|
| `x` | Position on the x-axis |
| `y` | Position on the y-axis |
| `colour` | Outline or point colour |
| `fill` | Fill colour (bars, boxes, ribbons) |
| `size` | Size of points or line thickness |
| `shape` | Point shape (circle, triangle, square…) |
| `linetype` | Solid, dashed, dotted line |
| `alpha` | Transparency (0 = invisible, 1 = opaque) |

> **Important distinction:** Aesthetics inside `aes()` are *mapped from a variable*. Values set *outside* `aes()` are fixed constants — for example `colour = "blue"` applies the same blue to everything and does not reflect any data.

```r
ggplot(data = penguins,
       aes(x = bill_length_mm,
           y = bill_depth_mm))
```

Now we have axes with labels and scales. ggplot knows *where* to put things — but still nothing is drawn. We need a geometry layer.

---

### Layer 3: Geoms — the marks that represent the data

**Geoms** (geometric objects) are the **visual marks** that represent the data. They answer: *how* should the data be drawn?

Every `geom_*()` function adds a new layer on top of the plot. Some of the most common geoms are:

| Geom | What it draws |
|---|---|
| `geom_point()` | Scatter plot points |
| `geom_line()` | Lines connecting observations |
| `geom_bar()` | Bar chart (counts rows automatically) |
| `geom_col()` | Bar chart (uses pre-calculated values) |
| `geom_histogram()` | Histogram (bins a numeric variable) |
| `geom_density()` | Smoothed density curve |
| `geom_boxplot()` | Box and whisker plot |
| `geom_violin()` | Violin plot |
| `geom_smooth()` | Trend or regression line |
| `geom_text()` | Text labels |

Geoms are added with the `+` operator — this is the **layer** system.

```r
ggplot(data = penguins,
       aes(x = bill_length_mm,
           y = bill_depth_mm)) +
  geom_point()
```

We now have a scatter plot. Three layers, three pieces of information:

1. `ggplot(data)` — the data
2. `aes(x, y)` — the mappings
3. `geom_point()` — the marks

---

### Layer 4: Aesthetic inheritance — global vs local

This is one of the most important concepts in ggplot2. Aesthetics defined inside `ggplot()` are **global** — they are **inherited** by every geom added afterwards. Aesthetics defined inside a specific `geom_*()` are **local** — they only apply to that one geom.

Here we add `colour = species` globally. Both `geom_point()` and `geom_smooth()` inherit it automatically.

```r
ggplot(data = penguins,
       aes(x      = bill_length_mm,
           y      = bill_depth_mm,
           colour = species)) +      # global — both geoms inherit this
  geom_point() +
  geom_smooth(method = "lm", se = FALSE)
```

Both layers are coloured by species. ggplot draws a separate regression line per species automatically because `colour` was declared globally.

Now compare this to moving `colour` *inside* `geom_point()` only:

```r
ggplot(data = penguins,
       aes(x = bill_length_mm,
           y = bill_depth_mm)) +
  geom_point(aes(colour = species)) +   # local to points only
  geom_smooth(method = "lm",
              se     = TRUE,
              colour = "black")          # fixed value, NOT inside aes()
```

Now the smooth line is a single global line covering all species combined. The colour only applies to the points. Notice that `colour = "black"` is *outside* `aes()` — it is a fixed constant, not a data mapping.

---

### Layer 5: A complete plot — labels and themes

We can keep adding layers indefinitely. `labs()` controls titles and axis labels. `theme_*()` controls the non-data appearance of the plot (fonts, background, gridlines, etc.).

```r
ggplot(data = penguins,
       aes(x      = bill_length_mm,
           y      = bill_depth_mm,
           colour = species,
           shape  = species)) +
  geom_point(size = 2.5, alpha = 0.8) +
  geom_smooth(method = "lm", se = FALSE, linewidth = 0.8) +
  labs(
    title    = "Bill Length vs Bill Depth",
    subtitle = "Palmer Penguins — by species",
    x        = "Bill Length (mm)",
    y        = "Bill Depth (mm)",
    colour   = "Species",
    shape    = "Species",
    caption  = "Data: palmerpenguins package"
  ) +
  theme_bw(base_size = 13)
```

**Summary of the layering system:**

| Layer function | What it controls |
|---|---|
| `ggplot()` | Canvas + data source |
| `aes()` | Variable → visual property mappings |
| `geom_*()` | The marks drawn on the canvas |
| `labs()` | Titles, axis labels, captions |
| `scale_*()` | How mappings translate to actual values |
| `theme_*()` | Non-data appearance (fonts, gridlines…) |
| `facet_*()` | Split into small multiple panels |

## Part 2 — Plot Types by Variable Type

The type of plot you should use depends on *how many variables* you have and *what type* they are. This section walks through each combination.

---

## 2A — One Numerical Variable
### *"What does the distribution of this variable look like?"*

When you have a single continuous variable, you want to understand its **shape**, **spread**, and **centre**.

---

### Histogram

A histogram bins the numeric variable into intervals and counts the observations in each bin. The `binwidth` argument controls how wide each bin is — experiment with different values to find the right level of detail.

```r
ggplot(data = penguins,
       aes(x = flipper_length_mm)) +
  geom_histogram(binwidth = 5,
                 fill     = "#4E9BB9",
                 colour   = "white") +
  labs(title = "Histogram — Flipper Length",
       x     = "Flipper Length (mm)",
       y     = "Count") +
  theme_bw()
```

Try `binwidth = 1` (too detailed) versus `binwidth = 20` (too coarse) to understand its effect. The right binwidth reveals the structure without noise.

We can add `fill = species` to reveal the mixture of distributions hidden in the combined histogram. Using `position = "identity"` overlaps the bars; `alpha` makes them transparent so all groups remain visible.

```r
ggplot(data = penguins,
       aes(x    = flipper_length_mm,
           fill = species)) +
  geom_histogram(binwidth  = 5,
                 position  = "identity",
                 alpha     = 0.6,
                 colour    = "white") +
  scale_fill_manual(values = c("#4E9BB9","#E07B54","#6AAB6E")) +
  labs(title = "Histogram by Species — Flipper Length",
       x     = "Flipper Length (mm)",
       y     = "Count",
       fill  = "Species") +
  theme_bw()
```

---

### Density Plot

A density plot is a smoothed, continuous version of the histogram. The y-axis shows **density** (the area under the curve sums to 1), not raw counts. This makes it much better than histograms for comparing the shapes of distributions across groups.

```r
ggplot(data = penguins,
       aes(x      = flipper_length_mm,
           fill   = species,
           colour = species)) +
  geom_density(alpha = 0.35, linewidth = 0.8) +
  scale_fill_manual(values   = c("#4E9BB9","#E07B54","#6AAB6E")) +
  scale_colour_manual(values = c("#3A7A99","#C05A33","#4A8B4E")) +
  labs(title  = "Density Plot — Flipper Length by Species",
       x      = "Flipper Length (mm)",
       y      = "Density",
       fill   = "Species",
       colour = "Species") +
  theme_bw()
```

**Histogram vs Density — when to use which:**

| | Histogram | Density |
|---|---|---|
| Y-axis | Raw count | Probability density |
| Shows sample size? | Yes | No |
| Comparing groups | Harder | Easier |
| Bin/bandwidth dependency | Yes (binwidth) | Yes (bw) |

## 2B — One Categorical Variable
### *"How many observations fall into each category?"*

---

### Bar Plot

`geom_bar()` counts rows automatically — you only need to supply `x`. `geom_col()` is used when you already have the y values pre-calculated.

```r
ggplot(data = penguins,
       aes(x    = species,
           fill = species)) +
  geom_bar(width = 0.6) +
  scale_fill_manual(values = c("#4E9BB9","#E07B54","#6AAB6E"),
                    guide  = "none") +
  labs(title = "Bar Plot — Number of Penguins per Species",
       x     = "Species",
       y     = "Count") +
  theme_bw()
```

Adding a second categorical variable via `fill` shows the composition within each bar. The `position` argument controls how the bars are arranged:

```r
ggplot(data = penguins,
       aes(x    = species,
           fill = island)) +
  geom_bar(position = "stack",   # try "dodge" or "fill"
           width    = 0.6) +
  labs(title = "Species Composition by Island",
       x     = "Species",
       y     = "Count",
       fill  = "Island") +
  theme_bw()
```

| `position` | Effect |
|---|---|
| `"stack"` | Stacked absolute counts |
| `"dodge"` | Side-by-side bars |
| `"fill"` | Proportional (0–1), good for comparing percentages |

Use `position = "fill"` with `scale_y_continuous(labels = scales::percent)` to produce a clean proportional bar chart:

```r
ggplot(data = penguins,
       aes(x    = species,
           fill = island)) +
  geom_bar(position = "fill", width = 0.6) +
  scale_y_continuous(labels = scales::percent) +
  labs(title = "Proportional Bar — Island per Species",
       x     = "Species",
       y     = "Proportion",
       fill  = "Island") +
  theme_bw()
```

## 2C — One Numerical + One Categorical Variable
### *"How does this measurement differ across groups?"*

---

### Box Plot

A box plot summarises the distribution of a numeric variable for each category using the **five-number summary**: minimum, lower quartile (Q1), median, upper quartile (Q3), and maximum. Points beyond 1.5 × IQR are plotted as individual outliers.

```r
ggplot(data = penguins,
       aes(x    = species,
           y    = body_mass_g,
           fill = species)) +
  geom_boxplot(width         = 0.5,
               outlier.shape = 21,
               outlier.size  = 2,
               alpha         = 0.7) +
  scale_fill_manual(values = c("#4E9BB9","#E07B54","#6AAB6E"),
                    guide  = "none") +
  labs(title = "Box Plot — Body Mass by Species",
       x     = "Species",
       y     = "Body Mass (g)") +
  theme_bw()
```

#### Always show your raw data

A box plot can hide small sample sizes, multimodal distributions, and other structure. It is good practice to always overlay the raw points. Set `outlier.shape = NA` to suppress the boxplot's own outlier points so they are not duplicated by the jitter layer.

```r
ggplot(data = penguins,
       aes(x    = species,
           y    = body_mass_g,
           fill = species)) +
  geom_boxplot(width         = 0.5,
               outlier.shape = NA,
               alpha         = 0.6) +
  geom_jitter(aes(colour = species),
              width  = 0.15,
              size   = 1.8,
              alpha  = 0.7) +
  scale_fill_manual(values   = c("#4E9BB9","#E07B54","#6AAB6E"),
                    guide    = "none") +
  scale_colour_manual(values = c("#2E7A99","#C05A33","#3A8B3E"),
                      guide  = "none") +
  labs(title = "Box Plot + Raw Data — Body Mass by Species",
       x     = "Species",
       y     = "Body Mass (g)") +
  theme_bw()
```

---

### Violin Plot

A violin plot shows the **full distribution shape** of the data, not just the five-number summary. Wider sections indicate more observations at that value. It is particularly useful for detecting bimodal distributions that a box plot would hide entirely. Combining a violin with a thin inner box plot gives the best of both worlds.

```r
ggplot(data = penguins,
       aes(x    = species,
           y    = body_mass_g,
           fill = species)) +
  geom_violin(trim  = FALSE,
              alpha = 0.7) +
  geom_boxplot(width = 0.1,
               fill  = "white",
               alpha = 0.8) +
  scale_fill_manual(values = c("#4E9BB9","#E07B54","#6AAB6E"),
                    guide  = "none") +
  labs(title = "Violin + Box Plot — Body Mass by Species",
       x     = "Species",
       y     = "Body Mass (g)") +
  theme_bw()
```

## 2D — Two Numerical Variables
### *"Is there a relationship between these two measurements?"*

---

### Basic Scatter Plot

```r
ggplot(data = penguins,
       aes(x = bill_length_mm,
           y = flipper_length_mm)) +
  geom_point(colour = "#4E9BB9",
             size   = 2.5,
             alpha  = 0.7) +
  labs(title = "Scatter Plot — Bill Length vs Flipper Length",
       x     = "Bill Length (mm)",
       y     = "Flipper Length (mm)") +
  theme_bw()
```

---

### Scatter + Regression Line

`geom_smooth(method = "lm")` fits a linear regression on top of the scatter plot. Setting `se = TRUE` adds a 95% confidence interval ribbon around the line.

```r
ggplot(data = penguins,
       aes(x = bill_length_mm,
           y = flipper_length_mm)) +
  geom_point(colour = "#4E9BB9", size = 2.5, alpha = 0.7) +
  geom_smooth(method = "lm",
              se     = TRUE,
              colour = "#E07B54",
              fill   = "#E07B54",
              alpha  = 0.15) +
  labs(title = "Scatter + Linear Regression",
       x     = "Bill Length (mm)",
       y     = "Flipper Length (mm)") +
  theme_bw()
```

---

### Scatter with Colour by Group

Adding `colour = species` maps species to point colour. This reveals structure that was hidden in the combined plot — what looked like a single relationship is actually three separate ones. This is a classic example of **Simpson's Paradox**.

```r
ggplot(data = penguins,
       aes(x      = bill_length_mm,
           y      = flipper_length_mm,
           colour = species)) +
  geom_point(size = 2.5, alpha = 0.8) +
  geom_smooth(method = "lm", se = FALSE, linewidth = 0.9) +
  scale_colour_manual(values = c("#4E9BB9","#E07B54","#6AAB6E")) +
  labs(title  = "Scatter — Bill Length vs Flipper Length by Species",
       x      = "Bill Length (mm)",
       y      = "Flipper Length (mm)",
       colour = "Species") +
  theme_bw()
```

---

### Scatter with Colour AND Shape — Redundant Encoding

Using **both** `colour` and `shape` to represent the same variable is called **redundant encoding**. It makes the plot more accessible for colour-blind readers and more legible when printed in black and white. This is standard practice for journal figures.

```r
ggplot(data = penguins,
       aes(x      = bill_length_mm,
           y      = flipper_length_mm,
           colour = species,
           shape  = species)) +
  geom_point(size = 2.8, alpha = 0.85) +
  geom_smooth(method = "lm", se = FALSE, linewidth = 0.8) +
  scale_colour_manual(values = c("#4E9BB9","#E07B54","#6AAB6E")) +
  scale_shape_manual(values  = c(16, 17, 15)) +   # circle, triangle, square
  labs(title  = "Colour + Shape Encoding — Species",
       x      = "Bill Length (mm)",
       y      = "Flipper Length (mm)",
       colour = "Species",
       shape  = "Species") +
  theme_bw()
```

---

### Scatter Encoding Four Variables at Once

By mapping a fourth variable (`body_mass_g`) to `size`, we can display four variables in a single plot. Use this carefully — too many aesthetics at once can make a plot difficult to read.

```r
ggplot(data = na.omit(penguins),
       aes(x      = bill_length_mm,
           y      = flipper_length_mm,
           colour = species,
           shape  = species,
           size   = body_mass_g)) +
  geom_point(alpha = 0.75) +
  scale_colour_manual(values = c("#4E9BB9","#E07B54","#6AAB6E")) +
  scale_shape_manual(values  = c(16, 17, 15)) +
  scale_size_continuous(range = c(1, 6), name = "Body Mass (g)") +
  labs(title    = "Bill Length vs Flipper Length",
       subtitle = "Colour = Species | Shape = Species | Size = Body Mass",
       x        = "Bill Length (mm)",
       y        = "Flipper Length (mm)",
       colour   = "Species",
       shape    = "Species") +
  theme_bw()
```

## Part 3 — Faceting: Small Multiples

Faceting splits a single plot into **multiple panels**, one per level of a categorical variable. This is one of ggplot2's most powerful features for exploring multivariate data.

`facet_wrap()` wraps panels into rows and columns based on a single variable. `facet_grid()` arranges panels in a strict rows × columns grid based on two variables.

```r
# facet_wrap() — one variable, wrapped into rows/columns
ggplot(data = penguins,
       aes(x      = bill_length_mm,
           y      = bill_depth_mm,
           colour = sex)) +
  geom_point(size = 2, alpha = 0.8, na.rm = TRUE) +
  geom_smooth(method = "lm", se = FALSE, na.rm = TRUE) +
  facet_wrap(~ species, ncol = 3) +
  scale_colour_manual(values = c("female" = "#E07B54",
                                  "male"   = "#4E9BB9"),
                       na.translate = FALSE) +
  labs(title  = "Bill Dimensions by Species and Sex",
       x      = "Bill Length (mm)",
       y      = "Bill Depth (mm)",
       colour = "Sex") +
  theme_bw()
```

```r
# facet_grid() — rows AND columns both controlled explicitly
ggplot(data = na.omit(penguins),
       aes(x    = flipper_length_mm,
           fill = species)) +
  geom_histogram(binwidth = 5, colour = "white", alpha = 0.85) +
  facet_grid(sex ~ species) +    # rows = sex, columns = species
  scale_fill_manual(values = c("#4E9BB9","#E07B54","#6AAB6E"),
                    guide  = "none") +
  labs(title = "Flipper Length — Faceted by Species and Sex",
       x     = "Flipper Length (mm)",
       y     = "Count") +
  theme_bw(base_size = 11)
```

## Part 4 — Themes: Controlling Non-Data Appearance

Themes control everything that is *not* the data itself — fonts, background, gridlines, tick marks, legend position, and so on. ggplot2 ships with several built-in themes:

```r
base_plot <- ggplot(data = penguins,
                    aes(x      = bill_length_mm,
                        y      = body_mass_g,
                        colour = species)) +
  geom_point(size = 2.5, alpha = 0.8) +
  scale_colour_manual(values = c("#4E9BB9","#E07B54","#6AAB6E")) +
  labs(x = "Bill Length (mm)", y = "Body Mass (g)", colour = "Species")

base_plot + theme_grey()    # default grey background
base_plot + theme_bw()      # white background, grey gridlines
base_plot + theme_classic() # white background, no gridlines
base_plot + theme_minimal() # minimal chrome
base_plot + theme_dark()    # dark background
```

You can customise any element of the theme using `theme()`:

```r
base_plot +
  theme_bw(base_size = 13) +
  theme(
    plot.title         = element_text(face = "bold", size = 14),
    axis.title         = element_text(face = "bold"),
    legend.position    = "bottom",
    legend.title       = element_text(face = "bold"),
    panel.grid.minor   = element_blank(),
    strip.background   = element_rect(fill = "#EEF2F6")
  ) +
  labs(title = "Custom theme() adjustments")
```

## Part 5 — Combining Plots with patchwork

The `patchwork` package lets you combine multiple ggplot objects into a single publication-ready figure using simple `|` (side by side) and `/` (above/below) operators.

```r
p1 <- ggplot(na.omit(penguins),
             aes(x = flipper_length_mm, fill = species)) +
  geom_density(alpha = 0.5, colour = NA) +
  scale_fill_manual(values = c("#4E9BB9","#E07B54","#6AAB6E")) +
  labs(title = "A — Flipper Length Distribution",
       x = "Flipper Length (mm)", y = "Density", fill = "Species") +
  theme_bw(base_size = 11)

p2 <- ggplot(na.omit(penguins),
             aes(x = species, y = body_mass_g, fill = species)) +
  geom_violin(alpha = 0.7, trim = FALSE) +
  geom_boxplot(width = 0.1, fill = "white", alpha = 0.8) +
  scale_fill_manual(values = c("#4E9BB9","#E07B54","#6AAB6E"),
                    guide  = "none") +
  labs(title = "B — Body Mass by Species",
       x = "Species", y = "Body Mass (g)") +
  theme_bw(base_size = 11)

p3 <- ggplot(na.omit(penguins),
             aes(x      = bill_length_mm,
                 y      = flipper_length_mm,
                 colour = species,
                 shape  = species)) +
  geom_point(size = 2.2, alpha = 0.8) +
  geom_smooth(method = "lm", se = FALSE, linewidth = 0.7) +
  scale_colour_manual(values = c("#4E9BB9","#E07B54","#6AAB6E")) +
  scale_shape_manual(values  = c(16, 17, 15)) +
  labs(title  = "C — Bill Length vs Flipper Length",
       x      = "Bill Length (mm)",
       y      = "Flipper Length (mm)",
       colour = "Species", shape = "Species") +
  theme_bw(base_size = 11)

p4 <- ggplot(na.omit(penguins),
             aes(x = species, fill = island)) +
  geom_bar(position = "fill", width = 0.6) +
  scale_y_continuous(labels = scales::percent) +
  labs(title = "D — Island Composition per Species",
       x = "Species", y = "Proportion", fill = "Island") +
  theme_bw(base_size = 11)

# Combine: top row = p1 + p2, bottom row = p3 + p4
final <- (p1 | p2) / (p3 | p4) +
  plot_annotation(
    title    = "Palmer Penguins — ggplot2 Lecture Summary",
    subtitle = "Density · Violin · Scatter · Proportional Bar",
    theme    = theme(
      plot.title    = element_text(face = "bold", size = 16),
      plot.subtitle = element_text(colour = "grey45", size = 11)
    )
  )

print(final)

ggsave("ggplot2_penguins_final_figure.png",
       plot = final, width = 14, height = 10, dpi = 300)
ggsave("ggplot2_penguins_final_figure.pdf",
       plot = final, width = 14, height = 10, device = "pdf")
```

## Quick Reference Summary

| Variable combination | Recommended plot | Geom |
|---|---|---|
| 1 numerical | Histogram | `geom_histogram()` |
| 1 numerical | Density plot | `geom_density()` |
| 1 categorical | Bar chart | `geom_bar()` |
| 1 numerical + 1 categorical | Box plot | `geom_boxplot()` |
| 1 numerical + 1 categorical | Violin plot | `geom_violin()` |
| 2 numerical | Scatter plot | `geom_point()` |
| 2 numerical + groups | Scatter with colour + shape | `aes(colour, shape)` |
| Any + split by category | Faceted version of above | `facet_wrap()` / `facet_grid()` |
